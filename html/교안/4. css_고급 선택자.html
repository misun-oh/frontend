FRAME 01

선택자란?

CSS 규칙을 "어떤 요소에" 적용할지 지정하는 문법

선택자(selector)는 HTML 문서 안에서 스타일을 적용할 대상을 찾아내는 규칙입니다. 크게 다섯 카테고리로 나눠서 이해하면 전체 그림이 잡힙니다.

카테고리예시의미
기본 선택자*, div, .card, #header태그/클래스/id로 직접 지정
결합자A B, A > B, A + B, A ~ B요소 사이의 관계로 지정
속성 선택자[type="text"], [href^="https"]HTML 속성값으로 지정
가상 클래스:hover, :nth-child(2), :not()상태나 순서로 지정
가상 요소::before, ::after, ::placeholder요소의 특정 부분/가상의 콘텐츠 지정

이 교안에는 곳곳에 선택자 테스터가 있어요. 입력창에 선택자를 직접 타이핑하면 아래 트리에서 매칭되는 요소가 파란색으로 강조됩니다.

FRAME 02

기본 선택자

전체(*) · 태그 · 클래스(.) · id(#)

선택자선택 대상
*문서의 모든 요소
div모든 <div> 태그
.cardclass="card"를 가진 모든 요소
#headerid="header"를 가진 요소 (문서에 유일해야 함)
Tip. 클래스는 여러 요소에 반복해서 붙일 수 있지만, id는 문서 전체에서 딱 하나의 요소만 가리켜야 합니다. 그래서 재사용 가능한 스타일은 클래스로, 고유한 지점(앵커, JS 훅)은 id로 쓰는 것이 관례입니다.

FRAME 03

결합자 (Combinator)

요소 사이의 "관계"로 대상을 지정

선택자기호의미
하위 결합자A B (공백)A 안에 있는 모든 하위 B (몇 단계든 상관없이)
자식 결합자A > BA의 바로 한 단계 아래 자식인 B만
인접 형제 결합자A + BA 바로 다음에 오는 형제 B 하나
일반 형제 결합자A ~ BA 뒤에 오는 모든 형제 B
Tip. 공백(하위 결합자)과 >(자식 결합자)를 가장 많이 헷갈립니다. 공백은 "몇 단계든 상관없이 안에 있으면 다 선택"이고, >는 "딱 한 단계 아래만" 선택합니다.

FRAME 04

속성 선택자

HTML 속성값을 기준으로 지정

선택자의미
[attr]해당 속성을 가진 요소 (값 무관)
[attr="value"]속성값이 정확히 일치
[attr^="value"]속성값이 value로 시작
[attr$="value"]속성값이 value로 끝남
[attr*="value"]속성값에 value가 포함

FRAME 06

가상 요소 (Pseudo-element)

요소의 특정 부분이나, 존재하지 않던 콘텐츠를 만들어 스타일링

선택자의미
::before요소의 콘텐츠 앞에 가상의 콘텐츠 삽입
::after요소의 콘텐츠 뒤에 가상의 콘텐츠 삽입
::placeholder입력창의 안내 문구(placeholder) 스타일링
::first-line문단의 첫 번째 줄만 스타일링
Tip. 콜론 하나(:hover)는 가상 클래스(상태), 콜론 두 개(::before)는 가상 요소(콘텐츠/부분)입니다. ::before/::after는 반드시 content 속성이 있어야 화면에 나타납니다.
::before로 따옴표 추가
가장 좋은 코드는 설명이 필요 없는 코드다
::after로 알림 개수 배지
🔔
::placeholder 스타일링
::first-line만 강조

이 문단의 첫 줄만 진하고 파란색으로 보이고, 그 다음 줄부터는 원래 스타일로 이어집니다. 문단이 길어져서 줄이 바뀌는 지점까지가 첫 줄로 인식됩니다.

.quote::before { content: "“"; } .badge::after { content: "3"; position: absolute; } input::placeholder { color: teal; font-style: italic; } p::first-line { font-weight: 700; }
FRAME 07

우선순위 (명시도, Specificity)

같은 요소에 여러 선택자가 겹치면, 어떤 스타일이 이길까?

여러 선택자가 같은 요소를 가리킬 때, 브라우저는 명시도(specificity)가 더 높은 선택자를 우선 적용합니다. 명시도는 세 단계 점수로 계산됩니다.

단계포함되는 것점수 단위
1단계id 선택자 (#header)가장 높음
2단계클래스, 속성, 가상 클래스 (.card, [type], :hover)중간
3단계태그, 가상 요소 (div, ::before)가장 낮음
Tip. 개수 비교는 자릿수 비교와 같습니다. id가 1개라도 있으면, 클래스나 태그가 아무리 많아도 절대 못 이깁니다 (인라인 스타일과 !important는 이 규칙 밖에서 더 강하게 작동합니다).

명시도 계산기 — 두 선택자를 비교해보세요

FRAME 10

실습 시 주의사항

선택자를 쓸 때 자주 하는 실수와 좋은 습관

이렇게 하세요
  • 스타일링은 클래스 위주로, id는 JS 훅이나 페이지 내 앵커용으로 아껴두기
  • 개발자 도구(F12)의 Elements/Styles 탭에서 실제로 어떤 규칙이 적용되고, 어떤 규칙이 취소선으로 밀려났는지 확인하는 습관 들이기
  • :hover뿐 아니라 :focus도 함께 스타일링해서 키보드 사용자를 배려하기
  • 선택자는 가능한 한 짧고 단순하게 — .card처럼 클래스 하나로 끝나는 게 이상적
이건 피하세요
  • 선택자를 5단계 이상 깊게 중첩하기 — div .wrap ul.list li a처럼 쓰면 HTML 구조가 조금만 바뀌어도 스타일이 깨짐
  • !important 남용 — 한 번 쓰기 시작하면 이후 모든 오버라이드가 !important 전쟁이 됨
  • div.card처럼 태그+클래스를 함께 쓰는 과잉 한정 — 특별한 이유가 없다면 .card만으로 충분
  • id 선택자로 스타일링하기 — 명시도가 지나치게 높아져서 나중에 다른 스타일로 덮어쓰기 어려워짐
Tip. 명시도가 헷갈릴 땐 "가장 단순한 선택자로도 원하는 요소를 정확히 짚어낼 수 있는가"를 먼저 고민해보세요. 복잡한 선택자가 필요하다면, 오히려 HTML에 클래스를 하나 더 추가하는 게 장기적으로 더 안전한 경우가 많습니다.
