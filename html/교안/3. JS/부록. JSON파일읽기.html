부록-1

JSON이란 무엇인가

⭐ 이 부록은 언제 다루면 좋을까요?

10장(외부API)의 10-1(JSON)을 배운 직후, 또는 fetch로 실제 데이터 파일을 다루기 직전에 20~25분 정도로 다루는 것을 추천합니다. movies.json 실습 파일이 함께 제공됩니다.

JSON(JavaScript Object Notation)은 자바스크립트 객체 표기법을 본떠 만든, 데이터를 저장하고 주고받기 위한 텍스트 형식입니다. 이름에 "JavaScript"가 들어있지만, 지금은 언어에 상관없이 거의 모든 프로그래밍 언어가 읽고 쓸 수 있는 데이터 교환의 표준입니다.

어디서 만나나요예시
서버가 보내주는 응답 데이터10장의 fetch로 받는 데이터가 대부분 JSON
설정 파일package.json처럼 프로그램 설정을 담는 파일
데이터 저장 파일지금 만들 movies.json처럼 목록형 데이터를 담는 파일
localStorage에 저장하는 값07장에서 객체를 저장할 때 JSON.stringify로 변환

겉모습은 자바스크립트 객체·배열과 거의 똑같아 보입니다. 아래는 실제 movies.json 파일의 일부입니다.

[
  {
    "title": "스텔라 오디세이",
    "genre": "SF · 액션",
    "rating": 9.2,
    "img": "https://picsum.photos/seed/movie1/300/400",
    "isNew": true
  },
  {
    "title": "도시의 밤",
    "genre": "느와르",
    "rating": 9.8,
    "img": "https://picsum.photos/seed/movie2/300/400",
    "isNew": false
  }
]
⭐ 파일 확장자가 .json이면, 안에 든 내용은 순수 텍스트입니다

movies.json을 메모장으로 열어도 그냥 글자입니다. 자바스크립트가 이 글자를 읽어서 진짜 배열·객체로 바꿔주는 과정이 이 부록의 핵심입니다.

부록-2

JSON 문법 규칙 — JS 객체와 비슷하지만 더 엄격합니다

JSON은 JS 객체 문법을 본떴지만, 훨씬 엄격한 규칙을 가진 순수 텍스트입니다. 아래 규칙을 하나라도 어기면 문법 오류가 납니다.

규칙JS 객체에서는JSON에서는
키(속성명)따옴표 생략 가능: {title: ...}반드시 큰따옴표: {"title": ...}
문자열 값큰따옴표·작은따옴표·백틱 다 가능반드시 큰따옴표만 (작은따옴표 불가)
마지막 항목 뒤 콤마있어도 되고 없어도 됨불가 — trailing comma는 오류
주석// 나 /* */ 사용 가능주석 불가 — 아예 못 씀
함수·undefined값으로 넣을 수 있음불가 — JSON은 순수 데이터만 표현
담을 수 있는 값모든 자료형문자열·숫자·불리언·null·배열·객체 만
🙋 학생들이 자주 틀리는 부분 — 자바스크립트 습관대로 JSON을 썼다가 문법 오류
❌ 이렇게 쓰면 안 됩니다 
// movies.json 파일에 이렇게 썼다면:
[
  {
    title: '스텔라 오디세이',   // 키에 따옴표 없음 + 작은따옴표 사용
    rating: 9.2,               // 마지막 줄 뒤에 콤마
  }
]
// JSON.parse 시도 시 → SyntaxError: Unexpected token 발생
✅ 이렇게 고칩니다 
[
  {
    "title": "스텔라 오디세이",
    "rating": 9.2
  }
]
// 키는 큰따옴표, 문자열 값도 큰따옴표, 마지막 항목 뒤 콤마 없음 ✅

JSON은 자바스크립트 객체가 아니라 텍스트이기 때문에, 자바스크립트 문법의 편의 기능(따옴표 생략, 주석, trailing comma)이 하나도 통하지 않습니다. 실무에서는 보통 에디터가 자동으로 맞춰주지만, 손으로 JSON 파일을 만질 땐 이 표를 기억하세요.

부록-3

JS 객체 vs JSON — 겉모습은 닮았지만 다른 존재

JS 객체JSON
정체메모리 안의 진짜 값 (자료형: object)순수 텍스트 (자료형: string)
만드는 방법{ title: "..." }처럼 코드로 직접파일에 글자로 저장하거나, 서버가 문자열로 전송
속성 접근movie.title 바로 가능불가능 — 먼저 객체로 변환해야 함
자바스크립트에서 쓰려면그대로 사용JSON.parse()로 객체로 변환한 뒤 사용
const jsonText = '{"title": "스텔라 오디세이", "rating": 9.2}';   // 이건 그냥 문자열!

console.log(typeof jsonText);        // "string"
console.log(jsonText.title);         // undefined — 문자열이라 .title 접근 불가!

const movie = JSON.parse(jsonText);  // 진짜 객체로 변환
console.log(typeof movie);           // "object"
console.log(movie.title);            // "스텔라 오디세이" — 이제 접근 가능 ✅
부록-4

복습 — JSON.stringify / JSON.parse

10장에서 배운 두 함수를 다시 정리합니다. 이름의 방향을 그대로 기억하면 헷갈리지 않습니다.

함수방향언제 쓰나
JSON.stringify(객체)객체 → 문자열(string)로객체를 파일에 저장하거나 서버로 보낼 때
JSON.parse(문자열)문자열 → 객체(object)로파일이나 서버에서 받은 JSON 텍스트를 사용할 때
const movie = { title: "겨울 왕좌", rating: 9.6 };

const text = JSON.stringify(movie);
console.log(text);          // '{"title":"겨울 왕좌","rating":9.6}'  (문자열)

const restored = JSON.parse(text);
console.log(restored.title); // "겨울 왕좌"  (다시 객체)
부록-5

실전 — fetch로 .json 파일을 읽어 JS 객체로 만들기

지금까지는 문자열을 직접 손으로 만들어 parse했지만, 실제로는 파일이나 서버에 있는 JSON을 fetch로 가져오는 경우가 대부분입니다. index.html과 같은 폴더에 movies.json 파일을 두고 읽어봅니다.

async function loadMovies() {
  try {
    const response = await fetch("movies.json");   // 같은 폴더의 파일 요청
    if (!response.ok) throw new Error("파일을 불러올 수 없습니다: " + response.status);

    const movies = await response.json();   // 응답 텍스트를 자동으로 JS 배열/객체로 변환!
    console.log(movies);          // 진짜 배열 — movies[0].title 처럼 바로 접근 가능
    console.log(movies.length);   // 10

    return movies;
  } catch (error) {
    console.log("로드 실패:", error.message);
  }
}

loadMovies();
⭐ response.json()이 fetch + JSON.parse를 한 번에 해줍니다

response.text()로 받으면 순수 문자열이 오지만, response.json()은 텍스트를 받아서 자동으로 JSON.parse까지 실행해줍니다. 그래서 fetch 코드에는 JSON.parse가 따로 안 보이는 것입니다 — 내부에서 대신 해주고 있을 뿐입니다.

이렇게 받은 movies 배열은 05장에서 만들었던 movies 객체 배열과 완전히 동일한 구조입니다. 08장의 renderMovies(movies)에 그대로 넘기면 바로 화면에 그려집니다 — 데이터의 출처(직접 작성 vs 파일 vs 서버)만 다를 뿐, 다루는 방법은 똑같다는 것이 이 과정 전체를 관통하는 원리입니다.

부록-6

흔한 실수 — 파일을 직접 열면 fetch가 실패합니다

🙋 학생들이 자주 틀리는 부분 — index.html을 더블클릭해서 열면 fetch("movies.json")이 막힘
❌ 이렇게 쓰면 안 됩니다 
// 파일 탐색기에서 index.html을 더블클릭해서 열면 주소창이 이렇게 됨:
// file:///C:/Users/.../index.html

fetch("movies.json")
  .then(response => response.json())
  .then(data => console.log(data));
// 콘솔 오류: Access to fetch has been blocked by CORS policy
// (file:// 프로토콜에서는 보안상 로컬 파일을 fetch로 읽는 것이 기본 차단됨)
✅ 이렇게 고칩니다 
// 해결: 반드시 "로컬 서버"를 통해 열어야 함
// 예: VS Code의 Live Server 확장 프로그램 사용
//     → 주소창이 http://127.0.0.1:5500/index.html 형태가 됨
// 이렇게 http://로 열린 상태에서는 fetch("movies.json")이 정상 동작 ✅

브라우저는 보안을 위해 file:// 프로토콜(파일을 그냥 연 상태)에서는 fetch 요청 상당수를 차단합니다. index.html이 movies.json 같은 파일을 fetch로 읽으려면, 반드시 Live Server 같은 도구로 http://를 통해 실행해야 합니다. "분명 코드는 맞는데 안 된다"는 상황의 8할이 이 문제입니다.
