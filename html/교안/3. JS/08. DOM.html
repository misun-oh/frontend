08. DOM

문서를 객체로 다루기 — 콘솔의 데이터가 드디어 화면이 되는 핵심 전환점

🎯 학습 목표

  • ☐ 노드 개념과 요소 노드/텍스트 노드를 구분할 수 있다
  • ☐ querySelector로 요소를 선택하고 textContent·style·classList를 바꿀 수 있다
  • ☐ createElement / appendChild와 innerHTML 두 방식으로 요소를 만들 수 있다
  • ☐ dataset으로 data-* 속성을 읽고 쓸 수 있다
  • ☐ 배열 데이터로 화면 목록을 동적으로 그릴 수 있다
08-1

DOM과 노드

DOM(Document Object Model)은 HTML의 태그를 객체화하여 자바스크립트에서 다룰 수 있게 한 것입니다. HTML을 트리로 구조화했을 때 각각의 태그가 노드입니다.

노드 종류의미
요소 노드(Element Node)태그 그 자체 (예: <h3>, <article>)
텍스트 노드(Text Node)태그 안에 기록된 문자 — 텍스트 노드를 갖는 태그(h1, p 등)와 갖지 않는 태그(img 등)가 있음
08-2

요소 선택하기

방법설명 · 리턴
document.getElementById('아이디')id로 1개 선택
document.querySelector('CSS선택자')CSS 선택자로 일치하는 첫 번째 요소 1개
document.querySelectorAll('CSS선택자')일치하는 모든 요소 (NodeList — forEach 가능)
const logo   = document.querySelector(".logo");     // 클래스
const header = document.querySelector("#site-header"); // 아이디
const cards  = document.querySelectorAll(".card");   // 전부 (6개)
console.log(cards.length);
⭐ CSS 시간에 배운 선택자를 그대로 씁니다

querySelector(".has-submenu > a")처럼 결합자·가상클래스까지 CSS 선택자 문법이 전부 동작합니다. CSS 실력이 곧 DOM 선택 실력입니다.

08-3

내용·속성·스타일 바꾸기

대상방법
텍스트요소.textContent = "새 글자" — 순수 텍스트로 취급 (안전)
HTML요소.innerHTML = "<b>굵게</b>" — 태그로 해석됨
일반 속성요소.setAttribute('src', 값) / 요소.getAttribute('src') / 요소.src = 값 (직접 대입)
data-* 속성요소.dataset.rating — data-rating을 읽고 씀 ⭐
스타일요소.style.backgroundColor = "gold"
클래스요소.classList.add / remove / toggle / contains
const card = document.querySelector(".card");

console.log(card.dataset.rating);        // data-rating="9.2" → "9.2"
card.style.borderColor = "gold";         // 스타일 직접 변경
card.classList.add("featured");          // 클래스 추가
⚠️ CSS 속성명의 - 는 카멜케이스로

자바스크립트 식별자에는 -를 쓸 수 없어 background-color → backgroundColor, font-size → fontSize로 바꿔 씁니다.

우리 index.html의 기존 스크립트가 이미 classList.toggle('open')으로 버거 메뉴를 여닫고 있죠 — 그 코드가 바로 이 절의 내용입니다.

08-4

요소 생성 ① — createElement 방식

텍스트 노드가 있는 요소: 요소 노드 생성 → 텍스트 노드 생성 → 요소에 텍스트 추가 → body(부모)에 요소 추가

const h3 = document.createElement("h3");        // 요소 노드 생성
const text = document.createTextNode("새 영화"); // 텍스트 노드 생성
h3.appendChild(text);                            // 요소에 텍스트 추가
document.body.appendChild(h3);                   // 문서에 추가

// 실무 단축: h3.textContent = "새 영화"; 로 텍스트 노드 생성을 대신

텍스트 노드가 없는 요소(img 등): 요소 생성 → 속성 설정 → 부모에 추가

const img = document.createElement("img");
img.setAttribute("src", "poster.jpg");   // 또는 img.src = "poster.jpg"
document.body.appendChild(img);
08-5

요소 생성 ② — innerHTML + 템플릿 리터럴 ⭐

여러 개의 요소를 한 번에 그릴 때는 HTML 문자열을 조립해 innerHTML에 대입하는 방식이 압도적으로 편합니다. 4~5장에서 만든 makeCard()가 드디어 화면과 만납니다.

const grid = document.querySelector(".grid");

let html = "";
movies.forEach(function (movie) {
  html += makeCard(movie);   // 5장 미션에서 만든 함수!
});
grid.innerHTML = html;        // 카드 6장이 한 번에 그려짐
⭐ 데이터가 바뀌면 다시 그린다

「movies 배열 → makeCard → innerHTML」 이 3단 파이프라인이 이 과정의 심장입니다. 9장의 검색 필터도, 10장의 API 데이터도 배열만 바꾸고 같은 파이프라인으로 다시 그리면 끝입니다.

⚠️ innerHTML과 보안(XSS)

사용자가 입력한 값을 검증 없이 innerHTML에 넣으면 악성 스크립트가 심어질 수 있습니다(XSS 공격). 사용자 입력을 표시할 때는 textContent를 쓰거나 값을 검증하세요. 우리가 직접 만든 데이터는 안전합니다.

08-6

요소 제거

방법설명
부모.removeChild(자식요소)부모에서 자식 노드를 제거 (전통 방식)
요소.parentNode.removeChild(요소)제거할 요소 기준으로 부모로 올라가 자신을 제거
요소.remove()자신을 바로 제거 (최신 방식, 가장 간단)
const firstCard = document.querySelector(".card");
firstCard.remove();                                // 최신 방식
// firstCard.parentNode.removeChild(firstCard);    // 전통 방식 (같은 결과)
