04. 함수

코드 묶음에 이름을 붙여 재사용하기 — 선언 방법 3가지와 호이스팅

🎯 학습 목표

  • ☐ 선언적 함수 · 익명 함수 · 즉시실행함수를 구분해서 작성할 수 있다
  • ☐ 매개변수와 return의 동작 규칙을 설명할 수 있다
  • ☐ 호이스팅 때문에 생기는 실행 순서 차이를 예측할 수 있다
  • ☐ 함수를 값처럼 다른 함수에 전달(콜백)할 수 있다
04-1

함수란

함수는 소스코드의 집합으로, 메서드·모듈·기능·프로시저 등으로도 불리며 자바스크립트에서는 function 자료형입니다. 「입력 값(매개변수) → 연산 → 반환값」 구조로, 같은 코드를 이름 하나로 몇 번이고 재사용할 수 있게 해줍니다.

04-2

함수 선언 3가지 방법

① 선언적 함수 — function 키워드에 함수 이름을 붙여 작성

function greet(name) {
  return `안녕하세요, ${name}님!`;
}
console.log(greet("지민"));
실행 결과
안녕하세요, 지민님!

② 익명 함수(함수 표현식) — 이름 없는 함수를 변수에 대입. 콜백 형태로 자주 사용

const greet2 = function (name) {
  return `반갑습니다, ${name}님!`;
};
console.log(greet2("수현"));

③ 즉시실행함수(IIFE) — 호출 없이 정의와 동시에 바로 실행

(function () {
  console.log("페이지 초기화 완료");
})();

참고로 ES6부터는 익명 함수를 더 짧게 쓰는 화살표 함수도 있습니다: const greet3 = (name) => `hi ${name}`; — 우리 index.html의 기존 스크립트가 이미 이 문법을 쓰고 있죠.

04-3

함수 호출 / 매개변수와 return

함수는 반드시 선언(정의)되어야 실행할 수 있고, 호출하는 것을 함수 실행이라 합니다. return 값이 있다면 받을 변수를 준비합니다.

함수명();                    // return 없는 경우
const result = 함수명(인자1, 인자2); // return 있는 경우
규칙동작
인자를 더 많이 넘기면초과분은 무시됨 (오류 아님)
인자를 더 적게 넘기면빠진 매개변수는 undefined가 됨
return 값을 지정하지 않으면undefined가 반환됨
return을 만나면즉시 함수를 종료하고 호출한 곳으로 돌아감
function add(a, b) { return a + b; }
console.log(add(1, 2, 99)); // 3        (99는 무시)
console.log(add(1));        // NaN      (b가 undefined → 1 + undefined)
04-4

가변인자 함수 — arguments

매개변수 개수가 정해지지 않은 함수를 가변인자 함수라고 합니다. 모든 (function 키워드) 함수 내부에는 arguments라는 유사 배열 객체가 자동 생성되어 전달된 인자가 순서대로 저장됩니다.

function sum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}
console.log(sum(1, 2, 3));       // 6
console.log(sum(10, 20, 30, 40)); // 100
⚠️ arguments는 '유사' 배열

length와 인덱스는 있지만 진짜 배열이 아니라서 push, sort 같은 배열 메서드는 쓸 수 없습니다. 또 화살표 함수에는 arguments가 없습니다. 최신 코드에서는 나머지 매개변수 function sum(...nums)를 씁니다.

04-5

함수를 매개변수로 전달 — 콜백

함수도 하나의 자료형(값)이므로 다른 함수의 매개변수로 전달할 수 있습니다. 전달받은 쪽에서 나중에 호출한다고 해서 콜백(callback) 함수라고 부릅니다.

function run(callback) {
  console.log("작업 시작");
  callback();              // 전달받은 함수를 호출
}

run(function () {
  console.log("전달된 함수 실행!");
});
실행 결과
작업 시작
전달된 함수 실행!
⭐ 이미 콜백을 쓰고 있습니다

3장의 sort(function(a,b){...}), forEach(function(item){...}), 그리고 9장의 addEventListener('click', function(){...}) — 전부 콜백입니다. 자바스크립트 전체를 관통하는 개념입니다.

04-6

내장 함수

함수설명
isNaN(number)number가 NaN인지 확인 — NaN 판별은 반드시 이 함수로 (NaN === NaN 은 false!)
isFinite(number)유한한 숫자인지 확인
eval(String)문자열을 자바스크립트 코드로 실행 — 보안 위험으로 사용 금지 수준의 레거시
⚠️ NaN은 자기 자신과도 다릅니다

NaN === NaN은 false입니다. 그래서 비교연산자가 아니라 isNaN()(또는 Number.isNaN())으로 판별해야 합니다.

04-7

호이스팅(Hoisting) — 선언이 끌어올려진다

자바스크립트는 코드를 실행하기 전에 선언부를 먼저 훑어 메모리에 등록합니다. 그래서 마치 선언이 파일 맨 위로 끌어올려진 것처럼 동작하는데, 이를 호이스팅이라고 합니다. 중요한 것은 선언 방식에 따라 결과가 다르다는 점입니다.

hello();          // 동작! — 선언적 함수는 "함수 전체"가 호이스팅
function hello() {
  console.log("안녕하세요");
}

bye();            // TypeError: bye is not a function
var bye = function () {
  console.log("안녕히가세요");
};
// var bye "선언"만 끌어올려져 undefined 상태 → undefined() 호출이라 오류
선언 방식호이스팅 결과선언 전 사용하면
function 선언적 함수함수 전체가 올라감정상 동작
var 변수선언만 올라감 (값은 undefined)undefined
let / const올라가지만 초기화 전 접근 금지(TDZ)ReferenceError
⭐ 호이스팅에 기대지 말 것

선언적 함수가 '아래에 있어도 동작'하는 건 알아두되, 코드는 항상 「선언 먼저, 사용 나중」 순서로 작성하는 습관을 들이세요. let/const가 선언 전 사용을 오류로 막아주는 이유가 바로 이것입니다.
