07. BOM

브라우저 자체를 다루는 객체들 — window · location · 타이머 · 저장소

🎯 학습 목표

  • ☐ BOM 구조와 window가 최상위 객체임을 설명할 수 있다
  • ☐ setTimeout / setInterval로 시간 기반 동작을 만들 수 있다
  • ☐ location 객체로 페이지 이동을 제어할 수 있다
  • ☐ localStorage에 데이터를 저장하고 꺼낼 수 있다
07-1

BOM(Browser Object Model) 구조

BOM은 브라우저 창 자체를 다루는 객체 모델입니다. 최상위에 window 객체가 있고, 그 아래에 location · navigator · history · screen, 그리고 문서를 다루는 document(=DOM, 8장)가 있습니다.

객체담당
window브라우저 창 — 자바스크립트의 최상위 객체. 모든 전역 객체·함수가 이 아래에 존재
screen사용자 모니터 화면 정보
location주소 표시줄(URL) 정보와 이동
navigator브라우저 자체 정보 (이름·버전·OS)
history방문 기록 — 앞으로/뒤로 가기
document문서(HTML) — 8장 DOM에서 학습

window가 최상위이므로 window.console.log() → console.log()처럼 window.는 생략 가능합니다.

07-2

window 객체 — 창 제어

window.open('주소', '이름 또는 open방식', '형태')로 새 창을 열 수 있습니다.

형태 옵션설명
width / height창의 너비 / 높이 값
location주소 입력 창 표시 여부
menubar / toolbar / status메뉴 / 툴바 / 상태표시줄 표시 여부
resizable화면 크기 조절 가능 여부 (yes | no | 1 | 0)
메서드설명
moveBy(x,y) / moveTo(x,y)창 위치 조정 (상대 / 절대)
resizeBy(x,y) / resizeTo(x,y)창 크기 조정 (상대 / 절대)
scrollBy(x,y) / scrollTo(x,y)스크롤 이동 (상대 / 절대)
focus() / blur()창에 초점 맞춤 / 제거
close()창 닫기 (스크립트로 연 창만 확실히 닫힘)
alert() / confirm() / prompt()경고 / 확인(true·false 리턴) / 입력(문자열 리턴) 대화상자
⚠️ 팝업 차단

요즘 브라우저는 사용자 클릭 없이 실행되는 window.open을 차단합니다. 새 창은 반드시 버튼 클릭 같은 사용자 동작 안에서 여세요.

07-3

타이머 — setTimeout · setInterval ⭐

메서드설명
setTimeout(함수, ms)일정 시간 후 한 번 실행 / 취소용 id 리턴
setInterval(함수, ms)일정 시간마다 반복 실행 / 취소용 id 리턴
clearTimeout(id)setTimeout 예약 취소
clearInterval(id)setInterval 반복 종료
setTimeout(function () {
  console.log("3초 후 한 번 실행");
}, 3000);

let count = 0;
const timerId = setInterval(function () {
  count++;
  console.log(count + "초 경과");
  if (count >= 5) clearInterval(timerId);  // 5초 뒤 반복 종료
}, 1000);
⭐ clearInterval을 위해 id를 꼭 변수에 저장

setInterval은 멈추라고 하기 전까지 영원히 돕니다. 리턴되는 id를 변수에 담아두지 않으면 멈출 방법이 없습니다. 11장의 캐러셀 자동 넘김이 setInterval, 마우스 올리면 멈추기가 clearInterval입니다.

07-4

window.onload — 문서가 준비된 후 실행

window.onload에 함수를 설정하면 모든 태그(이미지 포함)가 화면에 나타난 뒤 자동으로 실행됩니다. 1장에서 배운 「script를 body 끝에 두기」와 같은 문제(요소가 만들어지기 전 접근)를 푸는 또 다른 방법입니다.

window.onload = function () {
  console.log("문서 로드 완료 — 이제 어떤 요소든 안전하게 선택 가능");
};

참고: 최신 코드에서는 이미지까지 기다리지 않고 HTML 구조만 완성되면 실행되는 DOMContentLoaded 이벤트(9장)를 더 많이 씁니다.

07-5

screen · location · navigator · history

screen 속성설명
width / height화면 너비 / 높이
availWidth / availHeight작업표시줄 등을 뺀 실제 사용 가능 너비 / 높이
colorDepth / pixelDepth사용 가능한 색상 수 / 픽셀당 비트 수
location 속성설명 (예: https://movie.com:8080/list?genre=sf#top)
href전체 URL — 값을 대입하면 페이지 이동
protocol / host / hostname / porthttps: / movie.com:8080 / movie.com / 8080
pathname/list
search?genre=sf (요청 매개변수)
hash#top (앵커)
location 메서드설명
assign('주소')새 페이지 로드 — 뒤로 가기 가능
replace('주소')현재 페이지를 교체 — 뒤로 가기 불가
reload()현재 문서 새로고침

navigator는 브라우저 정보(userAgent 등), history는 방문 기록을 다룹니다: history.back()(뒤로), history.forward()(앞으로), history.go(-2)(2페이지 뒤로).

07-6

localStorage · sessionStorage — 브라우저 저장소

window의 저장소 객체로, 새로고침해도 데이터가 남습니다. localStorage는 지우기 전까지 영구, sessionStorage는 탭을 닫으면 삭제됩니다.

메서드설명
setItem(키, 값)저장 — 값은 문자열만 가능
getItem(키)읽기 — 없으면 null
removeItem(키)해당 키 삭제
clear()전부 삭제
localStorage.setItem("nickname", "무비덕후");
console.log(localStorage.getItem("nickname")); // 무비덕후

// 객체/배열은 JSON 문자열로 변환해서 저장 (JSON은 10장에서 자세히)
localStorage.setItem("fav", JSON.stringify({ title: "도시의 밤" }));
const fav = JSON.parse(localStorage.getItem("fav"));
console.log(fav.title);
⚠️ 문자열만 저장됩니다

객체를 그대로 setItem하면 "[object Object]"가 저장되어 버립니다. 반드시 JSON.stringify로 넣고 JSON.parse로 꺼내세요.
