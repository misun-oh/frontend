Q1

JSON 문법 오류 찾아 고치기

🌱 기초

아래 텍스트를 movies.json 파일에 저장하려고 합니다. 부록-2의 규칙표를 참고해 오류를 전부 찾아내세요.

주어진 코드/데이터
{
  title: '스텔라 오디세이',
  rating: 9.2,
  genres: ['SF', '액션'],
}
할 일
  • 이 텍스트에서 JSON 문법에 어긋나는 부분을 전부 찾아 표시한다 (최소 3곳)
  • 올바른 JSON으로 고쳐 쓴다
  • 고친 결과를 JSON.parse()로 실제 파싱해봐서 오류 없이 성공하는지 확인한다
💡 힌트
JSON은 자바스크립트 객체 리터럴의 '편의 문법'을 하나도 허용하지 않는다고 생각하면 쉽습니다.
✅ 정답 보기
// 오류 3가지:
// ① title, rating, genres — 키에 큰따옴표가 없음
// ② 'stella...', 'SF', '액션' — 작은따옴표 사용 (JSON은 큰따옴표만)
// ③ 마지막 속성(genres) 뒤에 불필요한 콤마

// 고친 버전
{
  "title": "스텔라 오디세이",
  "rating": 9.2,
  "genres": ["SF", "액션"]
}

// 검증
const text = '{"title":"스텔라 오디세이","rating":9.2,"genres":["SF","액션"]}';
console.log(JSON.parse(text));   // 오류 없이 객체로 파싱됨 ✅
Q2

stringify ↔ parse 왕복 연습

🌱 기초

부록-4를 참고해 객체와 JSON 문자열을 오가며 변환하세요.

할 일
  • title, rating, genres(배열) 속성을 가진 movie 객체를 만든다
  • JSON.stringify로 문자열로 바꾸고 typeof로 정말 문자열인지 확인한다
  • 그 문자열을 다시 JSON.parse로 객체로 되돌리고 movie.title이 잘 나오는지 확인한다
✅ 정답 보기
const movie = { title: "겨울 왕좌", rating: 9.6, genres: ["판타지", "어드벤처"] };

const text = JSON.stringify(movie);
console.log(typeof text);   // "string"

const restored = JSON.parse(text);
console.log(restored.title);       // "겨울 왕좌"
console.log(restored.genres[0]);   // "판타지"
Q3

movies.json 파일 읽어서 제목만 출력하기

🌿 응용

제공된 movies.json 파일을 index.html과 같은 폴더에 두고, fetch로 읽어보세요.

할 일
  • async function loadMovies() 함수를 부록-5 코드를 참고해 작성한다
  • fetch("movies.json")으로 파일을 요청하고 response.json()으로 배열을 얻는다
  • 받은 배열을 forEach로 순회하며 제목만 콘솔에 출력한다
  • 받은 배열의 length가 10인지 확인한다
  • Live Server 없이(파일 더블클릭으로) 열어서 무슨 오류가 나는지도 한 번 확인해본다 (부록-6)
💡 힌트
Live Server 확장 프로그램으로 index.html을 열어야 fetch가 정상 동작합니다 — 파일 더블클릭(file://)이면 CORS 오류가 납니다.
✅ 정답 보기
async function loadMovies() {
  try {
    const response = await fetch("movies.json");
    if (!response.ok) throw new Error("HTTP " + response.status);

    const movies = await response.json();
    movies.forEach(function (m) {
      console.log(m.title);
    });
    console.log(`총 ${movies.length}편`);   // 10편
  } catch (error) {
    console.log("로드 실패:", error.message);
  }
}

loadMovies();
Q4

movies.json으로 카드 렌더링까지

🎬 index.html 미션

파일에서 읽은 데이터를 08장의 renderMovies() 파이프라인에 그대로 연결해서, 하드코딩 없이 화면을 완성합니다.

할 일
  • loadMovies()가 fetch한 movies 배열을 return 하도록 만든다 (문제3의 함수를 이어서 사용)
  • loadMovies().then(function(movies){ renderMovies(movies); }) 형태로 연결한다 (또는 async 함수 안에서 await 후 바로 renderMovies 호출)
  • 화면에 10장의 카드가 movies.json 데이터로 그려지는지 확인한다
  • movies.json 파일의 아무 영화나 title을 직접 수정해 저장한 뒤 새로고침해서, 코드 수정 없이 화면 내용이 바뀌는지 확인한다
💡 힌트
코드는 그대로 두고 movies.json 파일의 데이터만 바꿔도 화면이 바뀐다는 것 — 이것이 '데이터와 화면을 분리해서 설계'하는 이유입니다.
✅ 정답 보기
async function loadMovies() {
  try {
    const response = await fetch("movies.json");
    if (!response.ok) throw new Error("HTTP " + response.status);
    return await response.json();
  } catch (error) {
    console.log("로드 실패:", error.message);
    return [];   // 실패 시 빈 배열 — renderMovies가 오류 없이 "카드 0개"로 처리됨
  }
}

// 8장에서 만든 renderMovies와 연결
loadMovies().then(function (movies) {
  renderMovies(movies);
  renderRanking(movies);
});
