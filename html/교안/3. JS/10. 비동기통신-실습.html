Q2

첫 fetch — then 체이닝

🌿 응용

테스트 API에서 회원 목록을 받아보세요.

할 일
  • fetch("https://jsonplaceholder.typicode.com/users") 를 호출한다
  • 첫 then에서 response.ok를 확인하고 response.json()을 리턴한다
  • 다음 then에서 각 회원의 name을 forEach로 출력한다
  • catch로 실패 메시지를 처리한다
  • 주소를 일부러 틀리게(/userszzz) 바꿔 404가 어떻게 처리되는지 관찰한다
✅ 정답 보기
fetch("https://jsonplaceholder.typicode.com/users")
  .then(function (response) {
    if (!response.ok) throw new Error("HTTP " + response.status);
    return response.json();
  })
  .then(function (users) {
    users.forEach(function (u) { console.log(u.name); });
  })
  .catch(function (error) {
    console.log("실패:", error.message);
  });

// /userszzz → response.ok가 false → throw → catch에서 "실패: HTTP 404"
// ok 확인이 없었다면 json 파싱 오류 등 엉뚱한 곳에서 터짐!
Q3

async/await로 다시 쓰기

🌿 응용

위 문제를 async/await + try/catch로 재작성하세요.

할 일
  • async function loadUsers() 를 선언한다
  • await fetch → response.ok 확인 → await response.json() 순서로 작성한다
  • try/catch로 감싸고, 회원 수를 `총 n명` 으로 출력한다
💡 힌트
await 한 줄 = then 한 단계. 두 방식이 1:1로 대응됨을 비교해보세요.
✅ 정답 보기
async function loadUsers() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/users");
    if (!response.ok) throw new Error("HTTP " + response.status);

    const users = await response.json();
    console.log(`총 ${users.length}명`);
  } catch (error) {
    console.log("실패:", error.message);
  }
}
loadUsers();
Q4

Promise 직접 만들기

🌿 응용

교안 10-5를 참고해 resolve/reject를 직접 다루는 Promise를 만드세요.

할 일
  • function checkAge(age) 함수를 만든다 — new Promise로 감싸서 리턴
  • age가 19 이상이면 resolve("입장 가능"), 미만이면 reject(new Error("19세 미만 입장 불가"))
  • checkAge(20).then(...) 과 checkAge(15).catch(...) 로 각각 테스트한다
💡 힌트
resolve는 성공 시 .then으로, reject는 실패 시 .catch로 값이 전달됩니다.
✅ 정답 보기
function checkAge(age) {
  return new Promise(function (resolve, reject) {
    if (age >= 19) {
      resolve("입장 가능");
    } else {
      reject(new Error("19세 미만 입장 불가"));
    }
  });
}

checkAge(20).then(function (msg) { console.log(msg); });        // "입장 가능"
checkAge(15).catch(function (err) { console.log(err.message); }); // "19세 미만 입장 불가"
Q5

Promise.all로 동시에 불러오기

🌳 도전

교안 10-8을 참고해 순차 처리와 동시 처리의 체감 속도 차이를 확인하세요.

할 일
  • async function loadSequential() — users, posts, todos를 await로 하나씩 순서대로 fetch한다
  • async function loadParallel() — 같은 3개를 Promise.all로 동시에 fetch한다
  • 각각 실행 전후로 console.time/console.timeEnd를 찍어 걸린 시간을 비교한다
  • 두 결과(users.length 등)가 서로 같은지도 확인한다
💡 힌트
네트워크 상황에 따라 차이 폭은 다르지만, 동시 처리가 순차 처리보다 느린 경우는 거의 없습니다.
✅ 정답 보기
async function loadSequential() {
  console.time("순차");
  const users = await fetch("https://jsonplaceholder.typicode.com/users").then(r => r.json());
  const posts = await fetch("https://jsonplaceholder.typicode.com/posts").then(r => r.json());
  const todos = await fetch("https://jsonplaceholder.typicode.com/todos").then(r => r.json());
  console.timeEnd("순차");
  console.log(users.length, posts.length, todos.length);
}

async function loadParallel() {
  console.time("동시");
  const [users, posts, todos] = await Promise.all([
    fetch("https://jsonplaceholder.typicode.com/users").then(r => r.json()),
    fetch("https://jsonplaceholder.typicode.com/posts").then(r => r.json()),
    fetch("https://jsonplaceholder.typicode.com/todos").then(r => r.json()),
  ]);
  console.timeEnd("동시");
  console.log(users.length, posts.length, todos.length);
}

loadSequential();
loadParallel();   // 보통 순차보다 눈에 띄게 빠름
Q6

POST 요청 흉내내기

🌿 응용

교안 10-9의 fetch 옵션(method/headers/body)을 연습합니다. jsonplaceholder는 POST를 받아도 실제 저장은 안 하고 흉내만 내주는 테스트 서버입니다.

할 일
  • fetch로 https://jsonplaceholder.typicode.com/posts 에 POST 요청을 보낸다
  • headers에 Content-Type: application/json을 설정한다
  • body에는 JSON.stringify({ title: "새 영화 등록", rating: 9.0 })를 담는다
  • await response.json()으로 응답을 받아 콘솔에 출력한다 (테스트 서버라 id가 새로 붙어서 돌아옴)
💡 힌트
headers를 빼먹으면 서버가 body를 못 읽을 수 있다는 것을 교안 10-9의 실수 사례로 복습하세요.
✅ 정답 보기
async function addMovie() {
  const response = await fetch("https://jsonplaceholder.typicode.com/posts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title: "새 영화 등록", rating: 9.0 }),
  });
  const data = await response.json();
  console.log(data);   // { title: "...", rating: 9.0, id: 101 } 같은 응답
}
addMovie();

공공데이터 응답 구조 탐험

🌳 도전

실제 공공 API(영화진흥위 박스오피스)를 호출해 응답 구조를 직접 파악하세요. (강사가 안내한 인증키 사용)

할 일
  • 교안 10-10의 URL로 어제 날짜 박스오피스를 요청한다
  • 먼저 console.log(data)로 전체 응답을 찍어 구조를 관찰한다
  • 일일 박스오피스 배열을 찾아 `1위 영화제목 (관객 n명)` 형태로 10줄을 출력한다
  • 인증키를 일부러 틀리게 넣고 어떤 응답이 오는지도 관찰한다
💡 힌트
응답이 JSON으로 잘 오는데 원하는 배열이 undefined라면, 경로(data.xxx.yyy)를 콘솔에서 한 단계씩 열어보세요.
✅ 정답 보기
const KEY = "강사_안내_인증키";
const url = `https://www.kobis.or.kr/kobisopenapi/webservice/rest/boxoffice/searchDailyBoxOfficeList.json?key=${KEY}&targetDt=20260712`;

async function loadBoxOffice() {
  try {
    const response = await fetch(url);
    if (!response.ok) throw new Error("HTTP " + response.status);
    const data = await response.json();

    console.log(data); // ① 구조 먼저 관찰!
    const list = data.boxOfficeResult.dailyBoxOfficeList;
    list.forEach(function (m) {
      console.log(`${m.rank}위 ${m.movieNm} (관객 ${m.audiCnt}명)`);
    });
  } catch (error) {
    console.log("실패:", error.message);
  }
}
loadBoxOffice();
// 키가 틀리면: HTTP는 200이지만 응답 안에 오류 코드가 담겨 옴
// → "HTTP 오류"와 "API 자체의 오류 응답"은 다르다는 것을 확인!
Q8

외부 데이터 카드 렌더링 + TTS 추천

🎬 index.html 미션

「fetch → 객체 배열 → renderMovies」 파이프라인을 완성하고, 오늘의 추천을 음성으로 읽어줍니다.

할 일
  • async function loadExternalMovies() — jsonplaceholder의 /photos?_limit=6 을 받아온다
  • 받은 배열을 map... 대신 forEach로 movies와 같은 구조({title, genre, rating, img, isNew})의 새 배열로 변환한다 (title ← photo.title.slice(0, 12), img ← photo.url, rating은 randomBetween(70, 99)/10, 한 개는 null)
  • 변환한 배열로 renderMovies()를 호출해 외부 데이터 카드가 화면에 뜨는지 확인한다
  • 도전(TTS): 버튼 클릭 시 6장의 pickTodayMovie() 결과를 교안 10-11 speak()로 읽어준다 — 키는 실습 후 즉시 폐기!
💡 힌트
renderMovies는 '어떤 배열이든' 그립니다. 데이터 출처(내 코드/서버)와 그리는 코드가 분리되어 있기 때문 — 이것이 함수로 쪼개온 보상입니다.
✅ 정답 보기
async function loadExternalMovies() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/photos?_limit=6");
    if (!response.ok) throw new Error("HTTP " + response.status);
    const photos = await response.json();

    const external = [];
    photos.forEach(function (photo, i) {
      external.push({
        title: photo.title.slice(0, 12),
        genre: "외부 데이터",
        rating: i === 5 ? null : Math.floor(Math.random() * 30 + 70) / 10,
        img: photo.url,
        isNew: true,
      });
    });

    renderMovies(external);   // 8장 파이프라인 재사용 — 이게 핵심!
  } catch (error) {
    document.querySelector(".grid").innerHTML =
      "<p>데이터를 불러오지 못했습니다 😢</p>";
  }
}
loadExternalMovies();
