Q1

영화 객체 만들기

🌱 기초

영화 한 편을 객체로 표현해보세요.

할 일
  • title, genre, rating 속성을 가진 movie 객체를 만든다
  • 점 표기법으로 제목을, 대괄호 표기법으로 평점을 출력한다
  • for...in으로 모든 속성을 `키 : 값` 형태로 출력한다
✅ 정답 보기
const movie = { title: "도시의 밤", genre: "느와르", rating: 9.8 };
console.log(movie.title);
console.log(movie['rating']);

for (const key in movie) {
  console.log(key + " : " + movie[key]);
}
Q2

속성 추가·삭제와 in 확인

🌱 기초

객체를 동적으로 수정해보세요.

할 일
  • rating 속성(8.7)을 추가한다
  • "rating" in movie 결과를 출력한다
  • delete로 rating을 삭제한 뒤 다시 in 결과를 출력한다
시작 코드
const movie = { title: "여름의 문" };
✅ 정답 보기
const movie = { title: "여름의 문" };
movie.rating = 8.7;
console.log('rating' in movie);  // true
delete movie.rating;
console.log('rating' in movie);  // false
Q3

this로 메서드 만들기

🌿 응용

객체 안에서 자기 속성을 사용하는 메서드를 작성하세요.

할 일
  • title, rating 속성과 intro 메서드를 가진 movie 객체를 만든다
  • intro는 `${this.title} (평점 ${this.rating})` 를 리턴한다
  • movie.intro() 결과를 출력한다
💡 힌트
메서드 안에서 자신의 속성을 부를 때는 반드시 this를 붙입니다.
✅ 정답 보기
const movie = {
  title: "파랑의 계절",
  rating: 9.5,
  intro: function () {
    return `${this.title} (평점 ${this.rating})`;
  },
};
console.log(movie.intro()); // 파랑의 계절 (평점 9.5)
Q4

생성자 함수 + prototype

🌳 도전

Movie 생성자를 만들고, 메서드는 prototype에 한 번만 저장하세요.

할 일
  • function Movie(title, genre, rating) 생성자를 작성한다 (this로 속성 저장)
  • Movie.prototype.getBadge 메서드를 추가한다 — rating이 있으면 "⭐ 평점", 없으면 "" 리턴(삼항연산자)
  • new로 영화 2개(하나는 rating null)를 만들어 getBadge() 결과를 비교한다
  • 두 객체의 getBadge가 같은 함수인지 === 로 확인한다
✅ 정답 보기
function Movie(title, genre, rating) {
  this.title = title;
  this.genre = genre;
  this.rating = rating;
}
Movie.prototype.getBadge = function () {
  return this.rating ? `⭐ ${this.rating}` : "";
};

const a = new Movie("도시의 밤", "느와르", 9.8);
const b = new Movie("은하수 특급", "SF", null);
console.log(a.getBadge());              // ⭐ 9.8
console.log(b.getBadge());              // (빈 문자열)
console.log(a.getBadge === b.getBadge); // true — prototype 공유!
Q5

movies 객체 배열 완성

🎬 index.html 미션

3장의 배열 2개(titles/ratings)를 객체 배열 하나로 재설계합니다. 이 데이터가 8~11장 내내 화면을 그리는 원본이 됩니다. (기존 titles/ratings 코드는 주석 처리)

할 일
  • main.js에 movies 객체 배열을 만든다 — 영화 6편, 각 영화는 title / genre / rating / img / isNew 속성
  • rating이 null인 개봉 예정작을 1편 포함한다 (배지 미션용!)
  • img는 "https://picsum.photos/seed/movie번호/300/400" 형태로 각기 다르게
  • forEach로 `제목 - 장르 - 평점` 6줄을 출력한다 (rating null이면 "개봉 예정")
  • 4장의 makeCard를 makeCard(movie) 형태(객체 하나 받기)로 수정하고, movies[0]으로 테스트한다
💡 힌트
매개변수 4개를 넘기던 것이 객체 하나로 바뀌면, 속성이 늘어나도 함수 시그니처를 안 바꿔도 됩니다.
✅ 정답 보기
// main.js — 이 데이터가 프로젝트의 원본!
const movies = [
  { title: "도시의 밤",     genre: "느와르",    rating: 9.8,  img: "https://picsum.photos/seed/movie1/300/400", isNew: false },
  { title: "파랑의 계절",   genre: "로맨스",    rating: 9.5,  img: "https://picsum.photos/seed/movie2/300/400", isNew: false },
  { title: "스텔라 오디세이", genre: "SF · 액션", rating: 9.2,  img: "https://picsum.photos/seed/movie3/300/400", isNew: true  },
  { title: "마지막 항해",   genre: "어드벤처",  rating: 9.0,  img: "https://picsum.photos/seed/movie4/300/400", isNew: false },
  { title: "여름의 문",     genre: "드라마",    rating: 8.7,  img: "https://picsum.photos/seed/movie5/300/400", isNew: false },
  { title: "은하수 특급",   genre: "SF",        rating: null, img: "https://picsum.photos/seed/movie6/300/400", isNew: true  },
];

movies.forEach(function (m) {
  const label = m.rating ? m.rating : "개봉 예정";
  console.log(`${m.title} - ${m.genre} - ${label}`);
});

// makeCard를 객체 하나를 받도록 수정
function makeCard(movie) {
  const badge = movie.rating
    ? `<span class="rating-badge">⭐ ${movie.rating}</span>`
    : "";
  const newBadge = movie.isNew ? `<span class="badge">NEW</span>` : "";
  return `
    <article class="card rounded-lg overflow-hidden flex-2" data-rating="${movie.rating}">
      ${badge}
      <img class="w-100 h-300px object-cover" src="${movie.img}" alt="${movie.title} 포스터">
      <div class="card-body bg-white px-3 py-5 fs-lg">
        <h3>${movie.title}</h3>
        <p class="genre text-gray">${movie.genre}</p>
        ${newBadge}
      </div>
    </article>`;
}
console.log(makeCard(movies[0]));
