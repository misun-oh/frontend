05-1

객체 선언과 속성 접근

객체는 키(속성명) : 값 쌍으로 정보를 묶는 자료형입니다. 중괄호로 생성하고, . 또는 [ ]로 접근합니다. 속성 값으로 모든 자료형이 올 수 있으며, 그중 함수인 속성을 메서드라고 부릅니다.

const movie = {
  title: "스텔라 오디세이",
  genre: "SF · 액션",
  rating: 9.2,
  play: function () {
    console.log(this.title + " 재생!");  // 객체 안에서 자신의 속성은 this로
  },
};

console.log(movie.title);       // 점 표기법
console.log(movie['rating']);   // 대괄호 표기법
movie.play();
실행 결과
스텔라 오디세이
9.2
스텔라 오디세이 재생!
⚠️ 특수한 속성명은 [ ]로만 접근

띄어쓰기·특수문자가 든 속성은 '개봉 연도': 2026처럼 따옴표로 선언하고, 접근도 movie['개봉 연도']처럼 대괄호로만 가능합니다.

05-2

속성 순회(for...in)와 in / with

객체의 모든 속성은 단순 for/while로는 출력할 수 없고 for...in문을 사용합니다.

for (const key in movie) {
  console.log(key + " : " + movie[key]);
}

in 키워드는 객체에 해당 속성이 있는지 확인합니다: 'rating' in movie → true

⚠️ with는 사용하지 않습니다

with(객체){ 속성명; }은 객체명을 생략하게 해주는 레거시 문법이지만, 스코프가 모호해져 strict mode에서는 금지되어 있습니다. '이런 게 있었다' 정도만 알아두세요.

05-3

구조분해할당 (Destructuring Assignment)

객체나 배열에서 필요한 값만 쏙쏙 꺼내 변수로 만드는 ES6 문법입니다. movie.title, movie.rating을 매번 따로 쓰는 대신, 한 줄로 동시에 꺼낼 수 있습니다.

const movie = { title: "스텔라 오디세이", genre: "SF", rating: 9.2 };

// 구조분해 없이
const title1 = movie.title;
const rating1 = movie.rating;

// 구조분해할당으로 — 객체의 속성명과 "똑같은 이름"으로 한 번에 꺼냄
const { title, rating } = movie;
console.log(title, rating);   // 스텔라 오디세이 9.2

다른 이름으로 받고 싶다면(별칭) 콜론(:)으로 새 이름을 지정합니다.

const { title: movieTitle, rating: movieRating } = movie;
console.log(movieTitle);   // 스텔라 오디세이

기본값도 매개변수 기본값(04-5)과 똑같은 방식으로 줄 수 있습니다 — 속성이 없거나 undefined일 때 사용됩니다.

const { title, ageRating = "전체 관람가" } = movie;   // movie엔 ageRating 속성이 없음
console.log(ageRating);   // "전체 관람가" (기본값 사용)

배열도 구조분해할 수 있습니다 — 이때는 이름이 아니라 순서(위치)대로 꺼냅니다.

const genres = ["SF", "액션", "모험"];
const [first, second] = genres;
console.log(first, second);   // SF 액션

함수의 매개변수 자리에서 바로 구조분해하면, 객체를 통째로 넘기고 함수 안에서 필요한 속성만 꺼내 쓸 수 있습니다.

function showMovie({ title, rating }) {   // 매개변수 자리에서 바로 구조분해
  console.log(`${title} - ${rating}`);
}
showMovie(movie);   // movie 객체를 통째로 넘기면 내부에서 자동으로 분해됨
🙋 학생들이 자주 틀리는 부분 — 객체 속성명과 다른 이름으로 그냥 꺼내려는 실수
❌ 이렇게 쓰면 안 됩니다 
const movie = { title: "스텔라 오디세이", rating: 9.2 };

const { movieTitle } = movie;   // movie엔 "movieTitle"이라는 속성이 없음!
console.log(movieTitle);        // undefined ❌
✅ 이렇게 고칩니다 
// 방법 1: 실제 속성명 그대로 사용
const { title } = movie;
console.log(title);             // "스텔라 오디세이" ✅

// 방법 2: 별칭 문법으로 이름을 바꿔서 받기
const { title: movieTitle } = movie;
console.log(movieTitle);        // "스텔라 오디세이" ✅

구조분해할당의 { } 안 이름은 객체의 실제 속성명과 정확히 일치해야 값을 꺼내옵니다. 다른 이름을 쓰고 싶다면 반드시 원래이름: 새이름 별칭 문법을 써야 합니다.

05-4

스프레드 연산자 (Spread Operator)

...(점 3개)로 배열이나 객체의 내용물을 낱개로 펼쳐서 복사하거나 합칠 때 씁니다. 3장에서 배열 복사에 잠깐 썼던 문법을 여기서 제대로 배웁니다.

① 배열 복사 — [...배열]은 같은 내용의 새 배열을 만듭니다 (03-4의 참조형 문제를 피하는 방법).

const original = ["도시의 밤", "파랑의 계절"];
const copy = [...original];   // 새 배열로 복사

copy.push("스텔라 오디세이");
console.log(original);   // ["도시의 밤", "파랑의 계절"]  ← 원본은 안 바뀜
console.log(copy);       // ["도시의 밤", "파랑의 계절", "스텔라 오디세이"]

② 배열 합치기 — 여러 배열을 펼쳐서 하나로 이어붙입니다.

const partA = ["SF", "액션"];
const partB = ["로맨스"];
const merged = [...partA, ...partB, "코미디"];
console.log(merged);   // ["SF", "액션", "로맨스", "코미디"]

③ 객체 복사 — {...객체}도 같은 원리로 새 객체를 만듭니다.

const movie = { title: "여름의 문", rating: 8.7 };
const copy = { ...movie };
copy.rating = 10;
console.log(movie.rating);   // 8.7  ← 원본은 안 바뀜

④ 객체 복사 + 속성 추가/덮어쓰기 — 실무에서 가장 많이 쓰는 패턴입니다. 뒤에 오는 속성이 앞의 같은 이름을 덮어씁니다.

const movie = { title: "어둠의 미로", genre: "SF", rating: 7.9 };

const bookmarked = { ...movie, isBookmarked: true };   // 속성 추가
console.log(bookmarked);   // {title:..., genre:..., rating:..., isBookmarked: true}

const rerated = { ...movie, rating: 9.9 };              // 기존 rating을 덮어씀
console.log(rerated.rating);   // 9.9
console.log(movie.rating);     // 7.9  ← 원본은 그대로
🙋 학생들이 자주 틀리는 부분 — 스프레드는 '얕은 복사' — 내부의 객체까지는 안 복사됨
❌ 이렇게 쓰면 안 됩니다 
const movie = { title: "도시의 밤", detail: { director: "김감독" } };
const copy = { ...movie };

copy.title = "다른 제목";           // 최상위 속성은 안전하게 분리됨
copy.detail.director = "다른 감독";  // 그런데 detail은...

console.log(movie.title);            // "도시의 밤" (안 바뀜, 정상)
console.log(movie.detail.director);  // "다른 감독" ❌ 원본도 바뀌어버림!
✅ 이렇게 고칩니다 
// detail처럼 객체 안에 또 객체가 있으면, 그 안쪽도 각각 스프레드해야 함
const copy = { ...movie, detail: { ...movie.detail } };

copy.detail.director = "다른 감독";
console.log(movie.detail.director);  // "김감독" ✅ 원본 안전

스프레드 연산자는 바로 한 겹만 복사합니다(얕은 복사). 객체 안에 또 객체·배열이 중첩되어 있으면, 그 안쪽은 여전히 원본과 같은 것을 참조합니다. 이 과정의 movies 데이터처럼 중첩 없이 평평한 구조로 설계하면 이 문제를 애초에 피할 수 있습니다.

05-5

속성 동적 추가/삭제 · 객체 배열

const movie = { title: "여름의 문" };

movie.rating = 8.7;                 // 속성 추가
movie.play = function () {          // 메서드 추가
  console.log("재생!");
};
delete movie.rating;                // 속성 삭제

생성된 객체를 배열에 넣으면 객체 배열이 됩니다. 3장에서 배열 2개(titles, ratings)로 나눠 담던 불편함이 여기서 해결됩니다.

const movies = [];
movies.push({ title: "도시의 밤", rating: 9.8 });
movies.push({ title: "파랑의 계절", rating: 9.5 });

console.log(movies[0].title);    // 도시의 밤
movies.forEach(function (m) {
  console.log(m.title, m.rating);
});
⭐ 객체 배열 = 이 과정 데이터의 최종 형태

영화 1편 = 객체 1개, 영화 목록 = 객체 배열. 이후 8장 렌더링, 9장 필터, 10장 API 응답까지 전부 이 구조로 다룹니다.

05-6

함수로 객체 만들기 vs 생성자 함수

① 팩토리 방식 — 함수가 객체를 만들어 리턴

function createMovie(title, rating) {
  const movie = {
    title: title,
    rating: rating,
  };
  return movie;
}
const m1 = createMovie("도시의 밤", 9.8);

② 생성자 함수 — this로 속성을 만들고 new 키워드로 객체를 생성. 생성자 이름은 대문자로 시작하고, instanceof로 어떤 생성자로 만들어졌는지 확인할 수 있습니다.

function Movie(title, rating) {
  this.title = title;
  this.rating = rating;
}

const m2 = new Movie("파랑의 계절", 9.5);
console.log(m2.title);
console.log(m2 instanceof Movie);  // true

new 생성자함수(...)는 단순히 함수를 부르는 게 아니라, 그 순간 4가지 일을 자동으로 해줍니다.

단계new가 하는 일
①빈 객체를 새로 만든다
②그 객체의 프로토타입을 생성자 함수의 prototype에 연결한다 (05-7의 prototype 공유가 여기서 시작됨)
③생성자 함수를 실행하되, this가 방금 만든 그 객체를 가리키게 한다
④함수가 따로 다른 객체를 return하지 않으면, 그 객체를 자동으로 리턴한다
function Movie(title) {
  console.log("this는:", this);   // ①②가 끝난 뒤라 이미 새 객체가 들어있음
  this.title = title;              // ③ this에 속성을 채움
}

const m = new Movie("도시의 밤");
console.log(m);   // Movie { title: "도시의 밤" }  ← ④ 자동으로 리턴된 객체
🙋 학생들이 자주 틀리는 부분 — new를 빼먹어서 undefined가 나오고 전역까지 오염됨
❌ 이렇게 쓰면 안 됩니다 
function Movie(title) {
  this.title = title;   // new 없이 부르면 this는 전역 객체(window)를 가리킴
}

const m = Movie("도시의 밤");   // new를 깜빡함!

console.log(m);            // undefined — return을 안 썼으니 당연히 undefined
console.log(m.title);      // TypeError: Cannot read properties of undefined

// 더 큰 문제: this.title = title 이 실행되면서
// title이 전역(window.title)에 새어나가 버림 — 코드 어디서든 접근 가능한 오염 발생
✅ 이렇게 고칩니다 
function Movie(title) {
  this.title = title;
}

const m = new Movie("도시의 밤");   // new를 붙임

console.log(m);         // Movie { title: "도시의 밤" } ✅
console.log(m.title);   // "도시의 밤" ✅

위 4단계(빈 객체 생성 → prototype 연결 → this 바인딩 → 자동 리턴)가 new가 있을 때만 일어납니다. new 없이 부르면 이 함수는 그냥 04장에서 배운 평범한 함수가 되어, return이 없으니 undefined를 돌려주고, this는 (비엄격 모드에서) 전역 객체를 가리켜서 의도치 않게 전역 변수를 만들어버립니다. 생성자 함수는 반드시 new와 함께 호출하세요.

05-7

prototype — 메서드를 한 번만 저장하기

두 방식의 결정적 차이는 중복 메서드 저장 방식입니다. 팩토리 방식은 객체를 만들 때마다 같은 메서드를 객체마다 복사해 저장하지만, 생성자는 prototype이라는 내부 객체에 메서드를 한 번만 저장하고 모든 객체가 공유합니다.

function Movie(title, rating) {
  this.title = title;
  this.rating = rating;
}

// prototype에 메서드 생성 → 모든 Movie 객체가 공유 (중복 저장 X)
Movie.prototype.getBadge = function () {
  return this.rating ? "⭐ " + this.rating : "";
};

const a = new Movie("도시의 밤", 9.8);
const b = new Movie("은하수 특급", null);
console.log(a.getBadge());  // ⭐ 9.8
console.log(b.getBadge());  // (빈 문자열)
console.log(a.getBadge === b.getBadge);  // true — 같은 함수를 공유!

참고: ES6의 class 문법은 이 생성자+prototype 구조를 보기 좋게 포장한 것입니다. class를 만나면 '아, 내부는 prototype이구나'라고 이해하면 됩니다. 다음 절에서 실제로 확인합니다.

05-8

class 문법 — 생성자+prototype을 보기 좋게 포장

class는 05-6·05-7에서 만든 생성자 함수 + prototype 패턴을 그대로 수행하면서, 더 읽기 쉬운 문법으로 포장한 것입니다. 완전히 새로운 기능이 아니라 같은 것을 다르게 쓰는 방법입니다.

class Movie {
  constructor(title, rating) {   // "생성자 함수" 부분
    this.title = title;
    this.rating = rating;
  }

  getBadge() {                   // prototype에 자동으로 들어가는 메서드
    return this.rating ? `⭐ ${this.rating}` : "";
  }
}

const a = new Movie("도시의 밤", 9.8);
console.log(a.getBadge());              // ⭐ 9.8
console.log(a instanceof Movie);         // true
console.log(typeof Movie.prototype.getBadge); // "function" — 진짜로 prototype에 들어있음!
05-6·05-7의 생성자 함수 방식class 방식
function Movie(title, rating) { this.title = title; ... }class Movie { constructor(title, rating) { this.title = title; ... } }
Movie.prototype.getBadge = function () {...}클래스 몸통 안에 getBadge() {...}만 쓰면 자동으로 prototype에 등록
new Movie(...)new Movie(...) (동일)
🙋 학생들이 자주 틀리는 부분 — 메서드 사이에 콤마를 붙이는 실수
❌ 이렇게 쓰면 안 됩니다 
class Movie {
  constructor(title) {
    this.title = title;
  },                    // 객체 리터럴 버릇으로 콤마를 붙임!

  getBadge() {
    return this.title;
  }
}
// SyntaxError: Unexpected token ','
✅ 이렇게 고칩니다 
class Movie {
  constructor(title) {
    this.title = title;
  }                     // 콤마 없음

  getBadge() {
    return this.title;
  }
}

class 안의 메서드들은 객체 리터럴({ })이 아니라 함수 선언들의 나열이라서, 05장 앞부분에서 배운 객체 리터럴과 달리 메서드 사이에 콤마를 쓰지 않습니다.

⚠️ class는 new 없이 부르면 아예 실행이 안 됩니다

05-6에서 생성자 함수를 new 없이 부르면 조용히 이상하게 동작했던 것과 달리, class는 new Movie(...) 없이 Movie(...)로만 불러도 바로 TypeError가 나서 실수를 즉시 알아챌 수 있습니다. 이런 안전장치 덕분에 최신 코드에서는 생성자 함수보다 class를 더 많이 씁니다.

05-9

this 정리 — 상황별로 가리키는 것

this는 고정된 값이 아니라, 함수를 "누가 어떻게 불렀는가"에 따라 매번 다시 결정되는 특수한 키워드입니다. 5가지 상황을 코드로 하나씩 확인합니다.

① 객체.메서드() 로 호출 — this는 점 앞에 있는 객체를 가리킵니다.

const movie = {
  title: "도시의 밤",
  play: function () {
    console.log(this.title);   // this === movie
  },
};
movie.play();   // "도시의 밤"

② new 생성자() 로 호출 — this는 새로 만들어지는 객체를 가리킵니다 (05-6의 4단계 참고).

function Movie(title) {
  this.title = title;   // this === 새로 만들어질 인스턴스
}
const m = new Movie("파랑의 계절");
console.log(m.title);   // "파랑의 계절"

③ 그냥 함수() 로 호출 — 점 앞에 아무도 없으므로, this는 전역 객체(window)를 가리킵니다. (엄격 모드 "use strict"에서는 undefined)

function plainFn() {
  console.log(this);   // window (일반 모드) 또는 undefined (strict 모드)
}
plainFn();

④ addEventListener의 function 핸들러 — this는 이벤트가 걸린 요소를 가리킵니다 (9장에서 자세히).

button.addEventListener("click", function () {
  console.log(this);   // this === button (클릭된 그 버튼)
});

⑤ 화살표 함수 — 화살표 함수는 자기만의 this가 없습니다. 자신을 감싸고 있는 바깥 함수의 this를 그대로 물려받습니다.

const movie = {
  title: "스텔라 오디세이",
  play: function () {
    const inner = () => {
      console.log(this.title);   // 화살표 함수라 바깥(play)의 this를 그대로 씀
    };
    inner();
  },
};
movie.play();   // "스텔라 오디세이"
🙋 학생들이 자주 틀리는 부분 — 메서드를 콜백으로 넘겼다가 this를 잃어버림
❌ 이렇게 쓰면 안 됩니다 
const movie = {
  title: "은하수 특급",
  play: function () {
    console.log(`${this.title} 재생 중`);
  },
};

movie.play();               // "은하수 특급 재생 중" — 정상

setTimeout(movie.play, 1000);
// TypeError 또는 "undefined 재생 중"
// movie.play를 함수 "이름만" 뚝 떼어서 넘기는 순간,
// "점 앞의 주인(movie)" 정보가 사라져버림 — ③번 규칙(그냥 함수 호출)이 적용됨
✅ 이렇게 고칩니다 
// 해결책 1: 화살표 함수로 감싸서 넘기기 (movie를 기억하는 함수를 새로 만듦)
setTimeout(() => movie.play(), 1000);   // 이 시점에 movie.play()로 "제대로" 호출됨

// 해결책 2: bind로 this를 미리 고정해서 넘기기
setTimeout(movie.play.bind(movie), 1000);

함수를 "실행"하지 않고 "이름만" 다른 곳에 전달하면(04-7의 콜백 전달과 같은 원리), this를 결정하던 "점 앞의 객체" 정보가 함께 딸려가지 않습니다. this가 필요한 메서드를 콜백으로 넘길 때는 항상 이 함정을 의심하세요 — 09장에서 이벤트 핸들러를 등록할 때도 자주 마주칩니다.

⭐ 헷갈리면 「점 앞을 보라」

movie.play()의 this는 movie. 「누가 이 함수를 호출했는가(점 앞의 주인)」가 this의 기본 규칙입니다. 화살표 함수만 예외로, 바깥 this를 물려받습니다. 점 앞이 아예 없다면(③, 콜백으로 뚝 떼어 전달된 경우) this는 원하는 대상을 잃어버립니다.
