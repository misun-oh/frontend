05-1

객체 선언과 속성 접근

객체는 키(속성명) : 값 쌍으로 정보를 묶는 자료형입니다. 중괄호로 생성하고, . 또는 [ ]로 접근합니다. 속성 값으로 모든 자료형이 올 수 있으며, 그중 함수인 속성을 메서드라고 부릅니다.

const movie = {
  title: "스텔라 오디세이",
  genre: "SF · 액션",
  rating: 9.2,
  play: function () {
    console.log(this.title + " 재생!");  // 객체 안에서 자신의 속성은 this로
  },
};

console.log(movie.title);       // 점 표기법
console.log(movie['rating']);   // 대괄호 표기법
movie.play();
실행 결과
스텔라 오디세이
9.2
스텔라 오디세이 재생!
⚠️ 특수한 속성명은 [ ]로만 접근

띄어쓰기·특수문자가 든 속성은 '개봉 연도': 2026처럼 따옴표로 선언하고, 접근도 movie['개봉 연도']처럼 대괄호로만 가능합니다.

05-2

속성 순회(for...in)와 in / with

객체의 모든 속성은 단순 for/while로는 출력할 수 없고 for...in문을 사용합니다.

for (const key in movie) {
  console.log(key + " : " + movie[key]);
}

in 키워드는 객체에 해당 속성이 있는지 확인합니다: 'rating' in movie → true

⚠️ with는 사용하지 않습니다

with(객체){ 속성명; }은 객체명을 생략하게 해주는 레거시 문법이지만, 스코프가 모호해져 strict mode에서는 금지되어 있습니다. '이런 게 있었다' 정도만 알아두세요.

05-3

속성 동적 추가/삭제 · 객체 배열

const movie = { title: "여름의 문" };

movie.rating = 8.7;                 // 속성 추가
movie.play = function () {          // 메서드 추가
  console.log("재생!");
};
delete movie.rating;                // 속성 삭제

생성된 객체를 배열에 넣으면 객체 배열이 됩니다. 3장에서 배열 2개(titles, ratings)로 나눠 담던 불편함이 여기서 해결됩니다.

const movies = [];
movies.push({ title: "도시의 밤", rating: 9.8 });
movies.push({ title: "파랑의 계절", rating: 9.5 });

console.log(movies[0].title);    // 도시의 밤
movies.forEach(function (m) {
  console.log(m.title, m.rating);
});
⭐ 객체 배열 = 이 과정 데이터의 최종 형태

영화 1편 = 객체 1개, 영화 목록 = 객체 배열. 이후 8장 렌더링, 9장 필터, 10장 API 응답까지 전부 이 구조로 다룹니다.

05-4

함수로 객체 만들기 vs 생성자 함수

① 팩토리 방식 — 함수가 객체를 만들어 리턴

function createMovie(title, rating) {
  const movie = {
    title: title,
    rating: rating,
  };
  return movie;
}
const m1 = createMovie("도시의 밤", 9.8);

② 생성자 함수 — this로 속성을 만들고 new 키워드로 객체를 생성. 생성자 이름은 대문자로 시작하고, instanceof로 어떤 생성자로 만들어졌는지 확인할 수 있습니다.

function Movie(title, rating) {
  this.title = title;
  this.rating = rating;
}

const m2 = new Movie("파랑의 계절", 9.5);
console.log(m2.title);
console.log(m2 instanceof Movie);  // true
⚠️ new를 빼먹으면

const m = Movie(...)처럼 new 없이 호출하면 this가 객체를 가리키지 않아 m은 undefined가 되고, 속성이 전역에 흘러나가는 사고가 납니다. 생성자는 반드시 new와 함께!

05-5

prototype — 메서드를 한 번만 저장하기

두 방식의 결정적 차이는 중복 메서드 저장 방식입니다. 팩토리 방식은 객체를 만들 때마다 같은 메서드를 객체마다 복사해 저장하지만, 생성자는 prototype이라는 내부 객체에 메서드를 한 번만 저장하고 모든 객체가 공유합니다.

function Movie(title, rating) {
  this.title = title;
  this.rating = rating;
}

// prototype에 메서드 생성 → 모든 Movie 객체가 공유 (중복 저장 X)
Movie.prototype.getBadge = function () {
  return this.rating ? "⭐ " + this.rating : "";
};

const a = new Movie("도시의 밤", 9.8);
const b = new Movie("은하수 특급", null);
console.log(a.getBadge());  // ⭐ 9.8
console.log(b.getBadge());  // (빈 문자열)
console.log(a.getBadge === b.getBadge);  // true — 같은 함수를 공유!

참고: ES6의 class 문법은 이 생성자+prototype 구조를 보기 좋게 포장한 것입니다. class를 만나면 '아, 내부는 prototype이구나'라고 이해하면 됩니다.

05-6

this 정리 — 상황별로 가리키는 것

호출 상황this가 가리키는 것
객체.메서드() 로 호출그 객체 (점 앞의 주인)
new 생성자() 로 호출새로 만들어지는 객체
그냥 함수() 로 호출window (전역 객체) — strict mode면 undefined
addEventListener의 function 핸들러이벤트가 걸린 요소 (9장)
화살표 함수자기만의 this가 없음 — 바깥의 this를 그대로 사용
⭐ 헷갈리면 「점 앞을 보라」

movie.play()의 this는 movie. 「누가 이 함수를 호출했는가(점 앞의 주인)」가 this의 기본 규칙입니다. 화살표 함수만 예외로, 바깥 this를 물려받습니다.
