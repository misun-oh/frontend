02-2

변수 — var, 그리고 let과 const

변수는 값을 담는 이름표입니다. 전통적으로 var로 선언했지만, 지금 작성하는 코드는 let/const가 표준입니다.

형식설명
변수명 = 값;var 없이 선언하면 전역변수로 간주 (window 객체의 멤버) — 사용 금지 습관!
var 변수명 = 값;함수 단위 유효범위. 재선언 허용(위험). 레거시 코드 읽을 때 필요
let 변수명 = 값;블록({ }) 단위 유효범위. 재선언 불가, 재할당 가능
const 변수명 = 값;블록 단위. 재선언·재할당 모두 불가. 선언과 동시에 값 필수
var a = 1;  var a = 2;   // var: 재선언해도 오류 없음 (버그의 원인)
let b = 1;  b = 2;       // let: 재할당 OK
const c = 1;
// c = 2;                // TypeError: Assignment to constant variable.
⭐ 이 과정의 규칙

기본은 const, 값이 바뀌어야 할 때만 let. var는 '읽을 줄만 알면' 됩니다. 레거시 교재·현장 코드에 var가 많으니 동작(재선언 허용, 함수 스코프)은 알아두세요.

명명 규칙 — ① 영문 대/소문자·숫자·_·$ 사용 가능 ② 첫 글자에 숫자 불가 ③ 예약어 불가 ④ 대소문자 구별 ⑤ 두 단어 결합 시 낙타봉 표기법(camelCase) 권장 (예: movieTitle) ⑥ 생성자 함수는 대문자로 시작(5장).

02-3

자료형이란

자료형(Data Type)은 값이 어떤 종류인지를 나타냅니다. 자바스크립트는 크게 원시 자료형(Primitive Type)과 참조 자료형(Reference Type) 두 그룹으로 나뉩니다.

구분포함하는 자료형특징
원시 자료형문자열, 숫자, 불리언, undefined, null값 자체를 변수에 직접 저장. 변수를 복사하면 값도 통째로 복사됨
참조 자료형객체, 배열, 함수값이 저장된 위치(주소)를 저장. 변수를 복사하면 같은 대상을 가리키게 됨 (5장에서 자세히)

① 문자열(String) — 따옴표(" ", ' ', 백틱 ` `)로 감싼 텍스트입니다.

const title = "스텔라 오디세이";
const genre = 'SF · 액션';

② 숫자(Number) — 자바스크립트는 정수/실수를 구분하지 않고 숫자는 전부 Number 하나입니다 (다른 언어의 int, float이 따로 없음).

const rating = 9;      // 정수
const avg = 9.2;       // 실수 — 둘 다 자료형은 똑같이 "number"

③ 불리언(Boolean) — true 또는 false, 딱 두 값만 가지는 자료형입니다. 조건문의 결과가 바로 이것입니다.

const isNewRelease = true;
const isSoldOut = false;

④ undefined — 값을 아직 할당하지 않은 상태를 뜻합니다. 변수를 선언만 하고 값을 안 넣으면 자바스크립트가 자동으로 넣어주는 값입니다 — 내가 직접 넣는 경우는 거의 없습니다.

let director;
console.log(director);   // undefined (선언만 하고 값을 안 넣음)

⑤ null — 값이 의도적으로 '없음'임을 내가 직접 표시할 때 씁니다. undefined와 비슷해 보이지만 성격이 다릅니다.

undefinednull
의미값이 아직 없음 (자동)값이 없다고 내가 명시 (수동)
누가 넣나자바스크립트가 자동으로개발자가 직접 대입
이 과정 사용처거의 안 씀 (변수 선언 직후 상태일 뿐)"평점 없음" 같은 의도적 빈 값을 표현 (05장 movies 데이터)
let director;              // 아직 선언만 함
console.log(director);     // undefined — "값이 있는지조차 모름"

let comingSoonRating = null; // "평점이 없다"는 것을 내가 확실히 표시
console.log(comingSoonRating); // null — "값이 없다는 걸 확실히 안다"

⑥ 객체(Object) — 다른 값들을 묶어서 담는 자료형입니다. 배열도, 함수도 자바스크립트 안에서는 전부 객체의 한 종류입니다 (자세한 내용은 03장 배열, 04장 함수, 05장 객체에서).

const movie = { title: "스텔라 오디세이", rating: 9.2 }; // 객체
const genres = ["SF", "액션"];                            // 배열도 객체
02-4

typeof로 자료형 확인하기

자바스크립트는 변수 선언 시 자료형을 지정하지 않기 때문에(동적 타입 언어), 지금 이 값이 무슨 자료형인지 코드로 직접 확인하려면 typeof 연산자를 씁니다.

typeof 결과예시
"string"typeof "스텔라 오디세이"
"number"typeof 9.2
"boolean"typeof true
"undefined"typeof 초기값없는변수
"object"typeof null , typeof { title: '...' } , typeof [1,2,3]
"function"typeof console.log
⚠️ typeof의 유명한 함정 2가지

① typeof null은 "object"가 나옵니다(언어 초기 설계의 오류가 표준으로 굳어짐 — null은 원시형인데 결과는 object로 나오는 유일한 예외). ② 배열도 "object"로 나오므로 배열인지 정확히 판별하려면 Array.isArray()를 씁니다.

🙋 학생들이 자주 틀리는 부분 — null과 undefined 혼동
❌ 이렇게 쓰면 안 됩니다 
// "평점이 아직 없는 개봉 예정작"을 표현하려는 코드
const movie = { title: "은하수 특급", rating: undefined };  // undefined를 직접 대입

if (movie.rating === null) {
  console.log("평점 없음");
} else {
  console.log("평점: " + movie.rating);   // 여기로 빠짐 — "평점: undefined" ❌
}
✅ 이렇게 고칩니다 
// "값이 의도적으로 없음"은 null로 명확히 표시
const movie = { title: "은하수 특급", rating: null };

if (movie.rating === null) {
  console.log("평점 없음");   // ✅ 의도대로 동작
} else {
  console.log("평점: " + movie.rating);
}

undefined는 "아직 값을 안 넣었다"는 자바스크립트의 자동 표시이고, null은 "값이 없다는 걸 개발자가 확정했다"는 의도적 표시입니다. 데이터를 직접 설계할 때(05장 movies처럼) '값 없음'을 표현하려면 undefined가 아니라 null을 명시적으로 대입하는 것이 이 과정의 규칙입니다.

02-5

문자열(String)과 자주 쓰는 메서드

문자열은 " "(큰따옴표) 또는 ' '(작은따옴표)로 감싼 텍스트입니다. 둘 중 아무거나 써도 되지만, 한 문자열 안에서는 같은 종류로 짝을 맞춰야 합니다.

const title1 = "스텔라 오디세이";
const title2 = '스텔라 오디세이';   // 결과는 동일

문자열 연결(옛날 방식): + 연산자로 문자열과 변수를 이어 붙일 수 있습니다.

const rating = 9.2;
const title = "스텔라 오디세이";
console.log("⭐ " + rating + " · " + title);
실행 결과
⭐ 9.2 · 스텔라 오디세이
⚠️ + 연결 방식의 불편함

변수가 늘어날수록 +와 따옴표가 계속 반복되어 어디서 문자열이 끝나고 변수가 시작하는지 읽기 어려워집니다. 따옴표 하나만 빠져도 오류가 나기 쉽습니다.

템플릿 리터럴(Template Literal, 백틱 사용) — ES6부터 생긴 문자열 작성법으로, 백틱(`)으로 감싸고 그 안에서 ${ } 자리에 변수나 계산식을 바로 넣을 수 있습니다.

const rating = 9.2;
const title = "스텔라 오디세이";

console.log(`⭐ ${rating} · ${title}`);        // 변수를 그대로 삽입
console.log(`평점 두 배: ${rating * 2}`);       // 계산식도 바로 삽입 가능!
실행 결과
⭐ 9.2 · 스텔라 오디세이
평점 두 배: 18.4
+ 연결 (옛날 방식)템플릿 리터럴 (백틱)
감싸는 기호" " 또는 ' '` ` (백틱, 키보드 숫자1 왼쪽)
변수 삽입"..." + 변수 + "..."`...${변수}...`
여러 줄 문자열\n을 직접 넣어야 함그냥 줄바꿔서 쓰면 됨
가독성변수가 많아지면 복잡글의 흐름을 그대로 유지

여러 줄 문자열도 템플릿 리터럴의 장점입니다 — 백틱 안에서는 실제로 줄바꿈한 그대로 출력됩니다.

const summary = `제목: ${title}
평점: ${rating}
한줄평: 올해 최고의 SF!`;
console.log(summary);
🙋 학생들이 자주 틀리는 부분 — 일반 따옴표에 ${ }를 쓰는 실수
❌ 이렇게 쓰면 안 됩니다 
const rating = 9.2;
console.log("평점: ${rating}");   // 큰따옴표를 씀!
// 출력: "평점: ${rating}"  ← 글자 그대로 출력됨, 계산 안 됨!
✅ 이렇게 고칩니다 
const rating = 9.2;
console.log(`평점: ${rating}`);   // 백틱을 씀
// 출력: "평점: 9.2"  ✅

${ } 문법은 백틱(`) 안에서만 동작합니다. 큰따옴표나 작은따옴표 안에 ${ }를 쓰면 그냥 글자 그대로 출력될 뿐 계산되지 않습니다 — 감싸는 기호를 백틱으로 바꿨는지 항상 확인하세요.

이 과정에서는 앞으로 변수를 문자열에 섞을 때 항상 템플릿 리터럴(백틱)을 기본으로 사용합니다.

메서드/속성설명

charAt(위치)해당 위치의 문자 리턴
substring(시작, 끝)일부분만 잘라 리턴 (끝 위치는 미포함)

02-7

데이터 형변환

형변환은 값의 자료형을 바꾸는 것으로, 명시적 형변환(내가 직접 함수로 바꿈)과 암묵적 형변환(연산자가 자동으로 바꿔줌) 두 가지가 있습니다. 암묵적 변환은 편리하지만 예상 못 한 결과의 원인이 되므로, 언제 어떻게 바뀌는지 정확히 알아야 합니다.

구분의미예시
암묵적 형변환연산자가 자동으로 바꿈 (내가 시키지 않음)"10" + 5 → "105" (자동으로 문자열화)
명시적 형변환함수로 내가 직접 바꿈Number("10") → 10

① 숫자 → 문자열: 숫자와 문자열을 + 연산하면 문자열이 우선되어 숫자가 문자열로 바뀝니다(암묵적, 연결). 명시적 변환은 String().

② 문자열 → 숫자: +를 제외한 사칙연산(-, *, /)에서는 숫자가 우선되어 문자열이 숫자로 바뀝니다(암묵적). 명시적 변환은 Number(), parseInt(), parseFloat().

console.log("10" + 5);        // "105"  (+는 문자열 연결이 우선! — 암묵적)
console.log("10" - 5);        // 5      (-는 숫자 연산 — 암묵적)
console.log(Number("9.2"));   // 9.2    (명시적)
console.log(parseInt("120분")); // 120   (숫자로 시작하면 읽을 수 있는 데까지)
console.log(Number("120분"));   // NaN   (Number는 전체가 숫자여야 함)
console.log(parseFloat("9.2점")); // 9.2

parseInt()는 인자를 2개 받을 수 있습니다 — 두 번째 인자로 진법(radix)을 지정하면 그 진법 기준으로 해석합니다.

console.log(parseInt("10"));      // 10  (기본은 10진수)
console.log(parseInt("10", 2));   // 2   (2진수 "10" = 십진수 2)
console.log(parseInt("FF", 16));  // 255 (16진수 "FF" = 십진수 255)
console.log(parseInt("777", 8));  // 511 (8진수 "777" = 십진수 511)

③ 불리언으로 변환: 명시적 변환은 Boolean(). 결과는 02-13에서 배울 falsy 6가지(false,0,"",null,undefined,NaN)는 false, 나머지는 전부 true 규칙을 그대로 따릅니다.

console.log(Boolean(0));      // false
console.log(Boolean(""));     // false
console.log(Boolean("0"));    // true   ⚠️ 문자열 "0"은 truthy!
console.log(Boolean(null));   // false
console.log(Boolean(123));    // true
console.log(Boolean([]));     // true   ⚠️ 빈 배열도 truthy!

④ 불리언 → 숫자 / 문자열: 반대 방향 변환도 있습니다. true는 1, false는 0으로 취급됩니다.

console.log(Number(true));    // 1
console.log(Number(false));   // 0
console.log(true + 1);        // 2      (true → 1로 암묵적 변환 후 덧셈)
console.log(String(true));    // "true" (문자열로)

⑤ toString() 메서드: 숫자·불리언 등에 직접 붙여서 문자열로 바꾸는 명시적 방법. 숫자에서는 진법을 인자로 줄 수도 있습니다.

console.log((255).toString());     // "255"
console.log((255).toString(16));   // "ff"  (16진수 문자열로)
console.log((10).toString(2));     // "1010" (2진수 문자열로)
⚠️ 형변환의 유명한 함정들 — 결과를 예측하기 어려운 경우

Number("") → 0 (빈 문자열은 0이지 NaN이 아님!)
Number(null) → 0 (null은 0)
Number(undefined) → NaN (undefined는 NaN — null과 다름!)
Number(" 9 ") → 9 (앞뒤 공백은 자동으로 무시됨)
"1" + 1 + 1 → "111" (첫 +부터 문자열 연결로 확정되면 뒤도 전부 문자열)
1 + 1 + "1" → "21" (숫자끼리 먼저 계산된 후에 문자열과 연결)

input 값은 항상 문자열: 나중에 검색창·폼에서 읽어오는 value는 무조건 문자열입니다. 숫자로 계산하려면 반드시 Number()/parseInt()로 변환하세요. "10" + 5 = "105" 사고가 실무 단골 버그입니다.

🙋 학생들이 자주 틀리는 부분 — 장바구니 합계 계산기
❌ 이렇게 쓰면 안 됩니다 
// 두 상품 가격을 입력창 2개에서 읽어와 합계를 구하려는 코드
const price1 = "5000";   // 첫 번째 input.value (문자열!)
const price2 = "3000";   // 두 번째 input.value (문자열!)

const total = price1 + price2;
console.log(`합계: ${total}원`);
// 기대: "합계: 8000원"
// 실제: "합계: 50003000원"  ← 두 문자열이 그냥 이어붙여짐!
✅ 이렇게 고칩니다 
const price1 = "5000";
const price2 = "3000";

const total = Number(price1) + Number(price2);  // 계산 전 반드시 변환!
console.log(`합계: ${total}원`);   // "합계: 8000원" ✅

덧셈(+)은 피연산자 중 하나라도 문자열이면 무조건 연결을 선택합니다. 곱셈(*)·뺄셈(-)은 자동으로 숫자 변환을 해주지만 +만 예외라서, +를 쓸 때는 반드시 숫자인지 직접 확인하는 습관이 필요합니다.

02-8

연산자의 종류와 역할 (개요)

연산자는 값을 계산하거나 비교하거나 저장하는 기호입니다. 역할에 따라 크게 5가지로 나뉘고, 이후 절에서 하나씩 자세히 다룹니다.

종류대표 기호하는 일
산술 연산자+ - * / %숫자를 계산한다 (사칙연산 + 나머지)
대입 연산자= += -= *= /= %=오른쪽 값을 왼쪽 변수에 저장한다
비교 연산자> < >= <= == != === !==두 값을 비교해 true/false를 만든다
논리 연산자&& || !여러 조건을 연결·반전한다
삼항 연산자조건 ? A : Bif/else를 한 줄로 표현한다
구분 (우선순위 높은 순)연산자설명
① 최우선( ) [ ] .괄호 · 배열 접근 · 속성 접근
② 단항++ -- ! +부호 -부호값 하나에만 적용
③ 산술* / % 그다음 + -곱셈류가 덧셈류보다 먼저
④ 비교> < >= <= == != === !==크기·동등 비교
⑤ 논리&& 그다음 ||AND가 OR보다 먼저
⑥ 삼항조건 ? 값1 : 값2if/else의 한 줄 표현
⑦ 대입= += -= *= /= %=가장 마지막에 수행
⭐ 우선순위를 다 외울 필요는 없습니다

헷갈리면 괄호 ( )를 쓰세요. a + b * c가 헷갈리면 a + (b * c)로 명확하게 쓰는 것이 실무에서도 더 좋은 습관입니다.

02-9

대입 연산자

대입 연산자는 오른쪽 값을 계산해서 왼쪽 변수에 저장합니다. =는 수학의 '같다'가 아니라 '담아라'는 명령이라는 점이 중요합니다 (같다는 비교 연산자인 === 로 표현).

연산자의미풀어쓰면
x = 5x에 5를 저장—
x += 5x에 5를 더해서 다시 저장x = x + 5
x -= 5x에서 5를 빼서 다시 저장x = x - 5
x *= 5x에 5를 곱해서 다시 저장x = x * 5
x /= 5x를 5로 나눠서 다시 저장x = x / 5
x %= 5x를 5로 나눈 나머지를 저장x = x % 5
let seats = 30;
seats = seats - 1;   // 기본형
seats -= 1;          // 복합 대입 (같은 결과, 더 짧게)
console.log(seats);  // 28
실행 결과
28
⚠️ = 와 == 와 === 는 완전히 다른 연산자

=(대입, 저장) / ==(느슨한 비교) / ===(엄격한 비교) — 셋을 헷갈려서 if (x = 5)처럼 쓰면 비교가 아니라 대입이 되어버리는 버그가 납니다. 조건식 안에서는 항상 ===를 쓰는 걸 규칙으로 삼으세요.

🙋 학생들이 자주 틀리는 부분 — 로그인 성공 여부 확인
❌ 이렇게 쓰면 안 됩니다 
// 아이디가 "admin"인지 확인하려는 코드
const userId = "guest";

if (userId = "admin") {        // = 하나! (대입)
  console.log("관리자 로그인!");
}
// userId가 "guest"인데도 "관리자 로그인!"이 출력됩니다
// → if(userId = "admin")은 userId에 "admin"을 대입해버리고,
//   대입된 결과("admin", truthy)를 조건으로 판단하기 때문!
✅ 이렇게 고칩니다 
const userId = "guest";

if (userId === "admin") {      // === 두 개! (비교)
  console.log("관리자 로그인!");
} else {
  console.log("일반 사용자");    // guest이므로 이게 출력됨 ✅
}

=(1개)은 저장, ===(3개)는 비교입니다. 조건식 괄호 안에 = 이 하나만 있다면 십중팔구 오타입니다 — 저장하면서 그 값 자체가 truthy라서 조건문이 항상 실행되는 것처럼 보이는, 매우 찾기 어려운 버그입니다.

02-10

비교 연산자

비교 연산자는 두 값의 크기나 동등함을 비교해서 결과를 항상 true 또는 false로 돌려줍니다. 조건문(02-14)의 재료가 됩니다.

연산자이름예시결과
>크다9 > 5true
<작다9 < 5false
>=크거나 같다5 >= 5true
<=작거나 같다4 <= 5true
==느슨한 비교 (값만 비교, 형변환 O)"5" == 5true ⚠️
!=느슨한 다름 비교"5" != 5false ⚠️
===엄격한 비교 (값+타입 비교) ⭐"5" === 5false
!==엄격한 다름 비교 ⭐"5" !== 5true
console.log(9.2 > 7.0);      // true
console.log("5" == 5);       // true  — 문자열 "5"를 숫자로 바꿔서 비교
console.log("5" === 5);      // false — 타입(문자열 vs 숫자)이 다름
console.log(9.2 !== 9.2);    // false
⭐ 이 과정의 규칙: 비교는 항상 === / !== 만 사용

==는 타입이 달라도 형변환해서 비교하기 때문에 예상 못 한 결과가 자주 나옵니다. === 와 !== 만 쓰면 이런 실수를 원천 차단할 수 있습니다.

🙋 학생들이 자주 틀리는 부분 — 장르 코드로 영화 걸러내기
❌ 이렇게 쓰면 안 됩니다 
// select의 value(문자열)와 영화 데이터의 genreId(숫자)를 비교하려는 코드
const selectedGenre = "1";        // <select>의 value는 항상 문자열!
const movie = { title: "스텔라 오디세이", genreId: 1 };  // 데이터는 숫자

if (selectedGenre === movie.genreId) {   // "1" === 1 → false!
  console.log("장르 일치!");
} else {
  console.log("장르 다름");   // 실제로는 같은 장르인데 여기로 빠짐 ❌
}
✅ 이렇게 고칩니다 
const selectedGenre = "1";
const movie = { title: "스텔라 오디세이", genreId: 1 };

if (Number(selectedGenre) === movie.genreId) {   // 비교 전에 타입을 맞춤
  console.log("장르 일치!");   // ✅ 정상 출력
}

===를 쓰는 것 자체는 옳았지만, 비교하려는 두 값의 타입이 애초에 다르면 === 도 항상 false를 반환합니다. "규칙: ===만 쓴다"에서 한 걸음 더 나아가, 비교 전에 두 값의 타입을 맞추는 습관이 필요합니다 — HTML의 select/input 값은 항상 문자열이라는 것을 특히 기억하세요.

02-11

논리 연산자

논리 연산자는 여러 개의 참/거짓을 연결하거나 뒤집습니다. 조건을 두 개 이상 합칠 때 사용합니다.

연산자이름동작 규칙
&&AND (그리고)둘 다 true일 때만 true
||OR (또는)하나라도 true면 true
!NOT (반전)true↔false를 뒤집음
ABA &amp;&amp; BA || B
truetruetruetrue
truefalsefalsetrue
falsetruefalsetrue
falsefalsefalsefalse
const rating = 9.2;
const isNewRelease = true;

console.log(rating >= 9.0 && isNewRelease);  // true && true  → true
console.log(rating >= 9.5 || isNewRelease);  // false || true → true
console.log(!isNewRelease);                  // false (반전)
⚠️ && 와 || 는 평가를 도중에 멈춥니다 (단락 평가)

false && 아무거나는 왼쪽이 false인 순간 오른쪽을 보지도 않고 바로 false입니다. true || 아무거나도 마찬가지로 오른쪽을 확인하지 않습니다. 그래서 user && user.name처럼 user가 없을 때 오류 없이 멈추게 하는 안전장치로도 자주 씁니다.

🙋 학생들이 자주 틀리는 부분 — 무료 관람 조건 판별 (괄호 누락)
❌ 이렇게 쓰면 안 됩니다 
// "SF 장르이면서 평점 9 이상" 이거나 "무료 쿠폰 보유"면 무료 관람
const genre = "드라마", rating = 6.0, hasCoupon = true;

const isFree = genre === "SF" && rating >= 9 || hasCoupon;
console.log(isFree);   // true — 의도는 맞았지만 우연일 뿐, 위험한 코드!
// &&가 ||보다 먼저 계산되어
// (genre==="SF" && rating>=9) || hasCoupon  으로 해석됨
// → 쿠폰만 있으면 장르·평점과 무관하게 항상 true가 되어버림
✅ 이렇게 고칩니다 
const genre = "드라마", rating = 6.0, hasCoupon = true;

// "괄호"로 원하는 그룹을 명확히 지정
const isFree = (genre === "SF" && rating >= 9) || hasCoupon;
console.log(isFree);   // 결과는 같지만, 의도가 코드에 명확히 드러남 ✅

이 예시는 결과가 우연히 같았지만, 조건이 3개 이상 섞이면 괄호 없이는 의도와 다르게 그룹지어지기 쉽습니다. &&와 ||를 함께 쓸 때는 항상 괄호로 그룹을 명시하는 것을 규칙으로 삼으세요 — 02-8 우선순위 표를 외우는 것보다 훨씬 안전합니다.

02-12

삼항 연산자

삼항연산자는 「조건 ? 참일 때 값 : 거짓일 때 값」 형태로, 아주 간단한 if/else를 한 줄로 줄여 씁니다.

const rating = 9.2;
const badge = rating ? `⭐ ${rating}` : "";  // 평점 있으면 배지, 없으면 빈 문자열
console.log(badge);
실행 결과
⭐ 9.2
⚠️ 삼항연산자는 '간단할 때만' — 조건이 복잡하면 if/else로

삼항연산자를 중첩해서 쓰면(a ? b ? 1 : 2 : 3) 읽기 매우 어려워집니다. 값 하나를 결정하는 간단한 상황에만 쓰고, 복잡한 분기는 02-14의 if/else를 쓰세요.

🔗 index.html 연결 포인트

이 삼항연산자 패턴이 그대로 카드 오른쪽 상단 평점 배지가 됩니다. 단, 이 코드가 왜 동작하는지 이해하려면 다음 절의 falsy 개념이 반드시 필요합니다 — rating이 ? 앞에 바로 오는 이유를 다음 절에서 설명합니다.

02-13

⚠️ Truthy와 Falsy — 조건식이 진짜로 보는 것

자바스크립트의 조건식(if, 삼항연산자, &&, ||)은 true/false가 아닌 값이 와도 동작합니다. 이때 자바스크립트는 그 값을 true 취급(truthy)할지 false 취급(falsy)할지 자동으로 판단합니다.

⭐ Falsy(거짓 취급) 값은 아래 6가지가 전부입니다 — 통째로 외우세요

false · 0 · "" (빈 문자열) · null · undefined · NaN
이 6개를 제외한 모든 값은 Truthy(참 취급)입니다. 문자열 "0", 빈 배열 [], 빈 객체 {}조차 Truthy입니다(글자·구조가 있으니까요) — 초보자가 가장 자주 틀리는 함정입니다.

if (0)   console.log("A"); else console.log("A는 falsy라 실행 안 됨");
if ("")  console.log("B"); else console.log("B는 falsy라 실행 안 됨");
if ("0") console.log("C, 문자열 '0'은 truthy!");   // 실행됨!
if ([])  console.log("D, 빈 배열도 truthy!");        // 실행됨!
if (null) console.log("E"); else console.log("E는 falsy라 실행 안 됨");
실행 결과
A는 falsy라 실행 안 됨
B는 falsy라 실행 안 됨
C, 문자열 '0'은 truthy!
D, 빈 배열도 truthy!
E는 falsy라 실행 안 됨

이제 02-12의 배지 코드를 다시 보면: rating ? A : B에서 rating이 9.2(truthy)면 A, rating이 0이나 null, undefined(falsy)면 B가 선택됩니다. rating !== null처럼 길게 쓰지 않고 값 자체를 조건에 바로 넣을 수 있는 이유가 바로 이 falsy/truthy 규칙입니다.

⚠️ 숫자 0은 falsy — '개봉 예정(평점 0)'과 '평점 없음(null)'을 구분해야 할 때 조심

평점이 진짜 0점인 영화가 있다면 rating ? A : B는 0도 falsy라 B로 빠집니다. 이런 경우엔 rating !== null && rating !== undefined처럼 명확한 비교를 써야 합니다. 이 과정의 movies 데이터는 '평점 없음'을 항상 null로만 표현하기로 약속해서 이 문제를 피합니다.

🙋 학생들이 자주 틀리는 부분 — 평점 0점 vs 평점 없음 구분
❌ 이렇게 쓰면 안 됩니다 
// 관객 혹평으로 실제 평점이 0.0점인 영화가 있다고 가정
const movie = { title: "최악의 영화", rating: 0 };

const badge = movie.rating ? `⭐ ${movie.rating}` : "평점 없음";
console.log(badge);   // "평점 없음" ❌ — 0점도 엄연히 평점이 매겨진 것인데!
// rating이 0(falsy)이라서 무조건 : 뒤로 빠져버림
✅ 이렇게 고칩니다 
const movie = { title: "최악의 영화", rating: 0 };

// truthy/falsy가 아니라 "값이 존재하는가"를 직접 물어봄
const badge = (movie.rating !== null && movie.rating !== undefined)
  ? `⭐ ${movie.rating}`
  : "평점 없음";
console.log(badge);   // "⭐ 0" ✅ — 0점이라는 사실이 정확히 표시됨

삼항연산자에 값을 그대로 넣는 패턴(rating ? A : B)은 편리하지만, 0이 의미 있는 값인 상황(평점, 개수, 순위 등)에서는 falsy 함정에 걸립니다. '값이 있다/없다'를 판단할 땐 truthy가 아니라 !== null 비교를 쓰는 습관이 안전합니다.

02-14

조건문 작성 방법

조건문은 조건에 따라 다른 코드를 실행합니다. 조건식 자리에는 비교/논리 연산자로 만든 값(또는 02-13의 truthy/falsy 값)이 들어갑니다.

① if — 조건이 참일 때만 실행

if (조건식) {
  // 조건식이 truthy일 때 실행
}

if (rating >= 9.0) {
  console.log("명작");
}

② if ~ else — 참/거짓 둘 중 하나

if (조건식) {
  // 참일 때
} else {
  // 거짓일 때
}

③ if ~ else if ~ else — 여러 조건을 순서대로 검사 (위에서부터 처음 맞는 조건 하나만 실행되고 나머지는 건너뜀)

const rating = 8.7;
if (rating >= 9.0) {
  console.log("명작");
} else if (rating >= 7.0) {
  console.log("볼만함");
} else {
  console.log("비추");
}
실행 결과
볼만함

④ switch — 하나의 값을 여러 경우와 비교할 때 if~else if를 대체. case마다 break로 반드시 끝내야 다음 case로 새지 않습니다.

switch (genre) {
  case "action":
    console.log("액션");
    break;              // 없으면 아래 case까지 줄줄이 실행됨!
  case "romance":
    console.log("로맨스");
    break;
  default:              // 어떤 case에도 안 맞을 때
    console.log("기타");
}
⚠️ switch의 break 누락은 실무 단골 버그

break를 빼먹으면 일치한 case부터 그 아래 모든 case가 조건 확인 없이 실행됩니다(fall-through). 의도한 게 아니라면 각 case 끝에 반드시 break를 쓰세요.

🙋 학생들이 자주 틀리는 부분 — 장르 이름 표시 (break 누락)
❌ 이렇게 쓰면 안 됩니다 
const genre = "action";
switch (genre) {
  case "action":
    console.log("액션");        // break 없음!
  case "romance":
    console.log("로맨스");      // break 없음!
  case "scifi":
    console.log("SF");
    break;
  default:
    console.log("기타");
}
// 출력: "액션" "로맨스" "SF"  — 3줄이 다 나와버림!
// case "action"이 맞는 순간, break가 없어서 아래 case들을 조건 확인 없이 계속 실행
✅ 이렇게 고칩니다 
const genre = "action";
switch (genre) {
  case "action":
    console.log("액션");
    break;                     // 여기서 확실히 탈출
  case "romance":
    console.log("로맨스");
    break;
  case "scifi":
    console.log("SF");
    break;
  default:
    console.log("기타");
}
// 출력: "액션"  ✅ 딱 한 줄만

switch를 처음 배울 때 가장 많이 하는 실수입니다. case를 쓸 때마다 반사적으로 break를 함께 쓰는 습관을 들이세요. (여러 case가 같은 동작을 공유해야 할 때만 의도적으로 break를 생략합니다 — 이건 심화 활용입니다)

02-15

반복문(제어문) 작성 방법

반복문은 같은 코드를 여러 번 실행합니다. 반복을 특정 조건까지로 제한하는 것이 핵심이라, '제어문'이라고도 부릅니다.

① for — 몇 번 반복할지 미리 알 때 가장 많이 씁니다. 소괄호 안 3부분을 각각 이해하는 것이 핵심입니다.

for (초기식; 조건식; 증감식) {
  // 조건식이 true인 동안 반복 실행
}
구성 요소역할이번 예시
초기식반복을 시작하기 전 딱 한 번만 실행let i = 1 (i를 1로 시작)
조건식매 반복 시작 전에 검사 — false면 반복 종료i <= 3 (3 이하인 동안)
증감식매 반복이 끝날 때마다 실행i++ (1씩 증가)
for (let i = 1; i <= 3; i++) {
  console.log(i + "번 좌석");
}
// 진행 순서: i=1(실행) → i++ → i=2(실행) → i++ → i=3(실행) → i++ → i=4는 조건 거짓이라 종료
실행 결과
1번 좌석
2번 좌석
3번 좌석

② while — 몇 번 반복할지 모르고, 조건이 유지되는 동안 계속할 때. 조건을 먼저 검사합니다.

while (조건식) {
  // 실행할 코드
  // 조건을 언젠가 false로 만드는 코드가 반드시 있어야 함!
}

let seats = 3;
while (seats > 0) {
  seats--;              // 이 줄을 빼먹으면 무한루프!
  console.log("예매 완료, 남은 좌석 " + seats);
}

③ do ~ while — 일단 한 번 실행하고 나서 조건을 검사. while과 달리 조건이 처음부터 거짓이어도 최소 1번은 실행됩니다.

let count = 0;
do {
  console.log("최소 1번은 실행:", count);
  count++;
} while (count < 0);   // 조건이 처음부터 거짓이지만 이미 1번 실행됨

④ break / continue — 반복 흐름 제어

키워드동작
break반복문을 즉시 완전히 종료하고 빠져나감
continue이번 회차만 건너뛰고 다음 반복으로 넘어감
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;   // 3만 건너뜀
  if (i === 5) break;      // 5에서 완전히 종료
  console.log(i);
}
// 출력: 1, 2, 4   (3은 건너뛰고, 5는 도달 전에 종료)
⚠️ while / do-while은 무한루프 주의

조건을 언젠가 false로 만드는 코드(예: seats--, count++)를 빼먹으면 브라우저가 멈춥니다. for문은 증감식이 강제로 보이는 위치에 있어 상대적으로 안전합니다.

🙋 학생들이 자주 틀리는 부분 — 배열을 for문으로 순회할 때 <= 실수
❌ 이렇게 쓰면 안 됩니다 
const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];   // length = 3

for (let i = 0; i <= titles.length; i++) {   // <= 를 씀 (실수!)
  console.log(titles[i]);
}
// "도시의 밤" "파랑의 계절" "스텔라 오디세이" undefined
// ↑ i가 3일 때 titles[3]은 존재하지 않아 undefined가 출력됨
// (인덱스는 0,1,2까지만 있는데 <=titles.length는 3까지 돌아버림)
✅ 이렇게 고칩니다 
const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];

for (let i = 0; i < titles.length; i++) {    // < 사용 (표준!)
  console.log(titles[i]);
}
// "도시의 밤" "파랑의 계절" "스텔라 오디세이"  ✅ 딱 3줄

배열의 인덱스는 0부터 (length-1)까지입니다. length 자체는 '개수'이지 '마지막 인덱스'가 아니라서, 배열을 순회하는 for문은 거의 항상 < titles.length를 씁니다(<= 가 아님). 좌석 번호(1번부터 n번까지)처럼 실제로 n까지 포함해야 하는 경우에만 <=를 씁니다 — 두 경우를 헷갈리지 않는 것이 핵심입니다.
