02-2

변수 — var, 그리고 let과 const

변수는 값을 담는 이름표입니다. 전통적으로 var로 선언했고, 원본 교재들도 var 기준으로 설명하지만, 지금 작성하는 코드는 let/const가 표준입니다.

형식설명
변수명 = 값;var 없이 선언하면 전역변수로 간주 (window 객체의 멤버) — 사용 금지 습관!
var 변수명 = 값;함수 단위 유효범위. 재선언 허용(위험). 레거시 코드 읽을 때 필요
let 변수명 = 값;블록({ }) 단위 유효범위. 재선언 불가, 재할당 가능
const 변수명 = 값;블록 단위. 재선언·재할당 모두 불가. 선언과 동시에 값 필수
var a = 1;  var a = 2;   // var: 재선언해도 오류 없음 (버그의 원인)
let b = 1;  b = 2;       // let: 재할당 OK
const c = 1;
// c = 2;                // TypeError: Assignment to constant variable.
⭐ 이 과정의 규칙

기본은 const, 값이 바뀌어야 할 때만 let. var는 '읽을 줄만 알면' 됩니다. 레거시 교재·현장 코드에 var가 많으니 동작(재선언 허용, 함수 스코프)은 알아두세요.

명명 규칙 — ① 영문 대/소문자·숫자·_·$ 사용 가능 ② 첫 글자에 숫자 불가 ③ 예약어 불가 ④ 대소문자 구별 ⑤ 두 단어 결합 시 낙타봉 표기법(camelCase) 권장 (예: movieTitle) ⑥ 생성자 함수는 대문자로 시작(5장).

02-3

자료형과 typeof

자바스크립트는 선언 시 자료형을 지정하지 않으므로, 자료형 확인에는 typeof 연산자를 사용합니다.

typeof 결과예시
"string"typeof "스텔라 오디세이"
"number"typeof 9.2
"boolean"typeof true
"object"typeof { title: '...' } , typeof [1,2,3]
"undefined"typeof 초기값없는변수
"function"typeof console.log
⚠️ typeof의 유명한 함정 2가지

① typeof null은 "object"가 나옵니다(언어 초기 설계의 오류가 표준으로 굳어짐). ② 배열도 "object"로 나오므로 배열 판별은 Array.isArray()를 씁니다.

02-4

문자열(String)과 자주 쓰는 메서드

문자열은 " " 또는 ' '로 묶인 값이며, ES6부터는 백틱( ` )을 쓰는 템플릿 리터럴로 변수를 문자열 안에 ${ }로 끼워 넣을 수 있습니다.

const title = "스텔라 오디세이";
const rating = 9.2;
console.log(`⭐ ${rating} · ${title}`);

숫자(number)와 기타 자료형

Math 메서드설명
Math.abs(x)절대값
Math.random()0 이상 1 미만의 난수 (소수)
Math.round(x)반올림

NaN(Not a Number)은 숫자가 아닌 데이터를 숫자처럼 연산했을 때 나오는 값입니다. 기타 자료형으로는 true/false 두 값을 갖는 논리값(Boolean), new로 만든 사용자 객체와 내장 객체를 포함하는 객체(Object), 선언되지 않았거나 값이 없는 undefined, 함수를 담는 function 자료형이 있습니다.

02-6

데이터 형변환

숫자 → 문자열: 숫자와 문자열을 + 연산하면 문자열이 우선되어 숫자가 문자열로 바뀝니다(연결). 강제 변환은 String().

문자열 → 숫자: +를 제외한 사칙연산(-, *, /)에서는 숫자가 우선되어 문자열이 숫자로 바뀝니다. 강제 변환은 Number(), parseInt(), parseFloat().

console.log("10" + 5);        // "105"  (+는 문자열 연결이 우선!)
console.log("10" - 5);        // 5      (-는 숫자 연산)
console.log(Number("9.2"));   // 9.2
console.log(parseInt("120분")); // 120   (숫자로 시작하면 읽을 수 있는 데까지)
console.log(Number("120분"));   // NaN   (Number는 전체가 숫자여야 함)
console.log(parseFloat("9.2점")); // 9.2
⚠️ input 값은 항상 문자열

나중에 검색창·폼에서 읽어오는 value는 무조건 문자열입니다. 숫자로 계산하려면 반드시 Number()/parseInt()로 변환하세요. "10" + 5 = "105" 사고가 실무 단골 버그입니다.

02-7

연산자 종류와 우선순위

구분 (우선순위 높은 순)연산자설명
① 최우선( ) [ ] .괄호 · 배열 접근 · 속성 접근
② 단항++ -- ! +부호 -부호값 하나에만 적용
③ 산술* / % 그다음 + -곱셈류가 덧셈류보다 먼저
④ 관계(비교)> < >= <= == != === !==크기·동등 비교
⑤ 논리&& 그다음 ||AND가 OR보다 먼저
⑥ 삼항조건 ? 값1 : 값2if/else의 한 줄 표현
⑦ 대입= += -= *= /= %=가장 마지막에 수행
⭐ == 말고 === 를 쓰세요

==는 형변환 후 비교해서 "5" == 5가 true가 됩니다. ===는 값과 자료형을 모두 비교합니다. 이 과정에서는 항상 === / !== 를 사용합니다.

삼항연산자는 「조건 ? 참일 때 값 : 거짓일 때 값」 형태로, 값이 있고 없음에 따라 다른 결과를 만들 때 매우 자주 씁니다.

const rating = 9.2;
const badge = rating ? `⭐ ${rating}` : "";  // 평점 있으면 배지, 없으면 빈 문자열
console.log(badge);

⭐ 9.2

02-8

제어문 — 조건문 · 반복문 · 분기문

조건문: if / if~else / if~else if~else / switch / 짧은 조건문(&&, ||)

const rating = 8.7;
if (rating >= 9.0) {
  console.log("명작");
} else if (rating >= 7.0) {
  console.log("볼만함");
} else {
  console.log("비추");
}
실행 결과
볼만함

반복문: for / while / do~while / for in (객체·배열 순회, 5장에서 자세히)

for (let i = 1; i <= 3; i++) {
  console.log(i + "번 좌석");
}

let seats = 3;
while (seats > 0) {
  seats--;              // 이 줄을 빼먹으면 무한루프!
  console.log("예매 완료, 남은 좌석 " + seats);
}

분기문: break(반복 즉시 종료), continue(이번 회차만 건너뜀)

⚠️ do~while은 최소 1번은 실행

while은 조건을 먼저 검사하지만, do~while은 일단 한 번 실행한 뒤 조건을 검사합니다.
