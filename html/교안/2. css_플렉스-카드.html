카드가 늘어날까, 늘려야 할까?

영화 추천 카드 리스트를 예제로, flexbox 반응형 레이아웃에서 이미지가 커지는 방식과 개수가 늘어나는 방식의 차이를 직접 비교합니다. 아래 데모 박스 오른쪽 아래 모서리를 드래그해서 너비를 직접 늘려보세요.

수업 질문 — 화면(부모 요소)의 너비가 커질 때, 자식 카드는 어떻게 반응해야 할까요?
① 카드 자체가 커진다   ② 카드 개수가 늘어난다   ③ 섞어서 쓴다
1카드 만들기 전에

flex 한 줄에 들어가는 3형제 — grow / shrink / basis

flex: 1 1 200px처럼 축약해서 쓰지만, 사실은 세 가지 값이 각각 다른 일을 합니다. 카드를 만들기 전에 이 세 속성을 먼저 손으로 만져보며 이해해요.

① flex-grow

남는 공간을 얼마나 나눠 가질지 정하는 비율. 0이면 안 늘어나고, 숫자가 클수록 더 많이 늘어남.

② flex-shrink

공간이 부족할 때 얼마나 줄어들지 정하는 비율. 0이면 절대 줄어들지 않음 (넘쳐도 유지).

③ flex-basis

시작크기(grow·shrink가 적용되기 전의 기본 크기). width의 대체 역할을 하는 시작점.

세 가지 방식 비교

방식CSS 핵심화면이 넓어지면이미지 왜곡적합한 상황
① 너비 확장 flex: 1 1 200px 카드가 계속 커짐 큼 (비율 이미지에 불리) 카드 수가 고정적이고 적을 때
② 개수 확장 flex: 0 0 190px + wrap 카드 개수만 늘어남 없음 포스터·썸네일처럼 비율이 중요한 콘텐츠
③ 하이브리드 (추천) flex: 1 1 220px; max-width: 280px 개수 먼저 늘고, 여백만 소폭 확장 거의 없음 대부분의 카드 리스트형 UI
6실습

학생 실습 과제

진행 순서

  1. 원본 예제의 .card-item에 flex: 1 1 200px만 적용하고 브라우저 창을 최대화해서 이미지가 어떻게 늘어나는지 관찰한다.
  2. 같은 요소에 flex: 0 0 190px + 부모에 flex-wrap: wrap을 적용해 차이를 비교한다.
  3. max-width를 추가했을 때와 뺐을 때의 차이를 각각 캡처해서 왜 실무에서 카드 크기를 제한하는지 서술한다.
  4. 심화: gap 값을 바꿔가며 카드 사이 간격이 줄바꿈 위치에 어떤 영향을 주는지 실험한다.
7보너스

aspect-ratio — 이미지 비율이 깨지는 걸 막는 속성

카드에 서로 다른 원본 비율의 이미지가 섞이면 카드 높이가 들쭉날쭉해집니다. aspect-ratio는 요소의 가로세로 비율을 고정해서 이 문제를 해결합니다.

img {
  width: 100%;
  aspect-ratio: 2 / 3;  /* 가로 2 : 세로 3 비율로 고정 */
  object-fit: cover;  /* 비율 유지하며 넘치는 부분은 잘라냄 */
}

width 값 하나만 정해줘도 height는 비율에 맞춰 자동으로 계산됩니다. 자주 쓰는 비율은 아래와 같아요.

1 / 1
정사각형(프로필)
16 / 9
와이드 영상
2 / 3
세로 포스터
4 / 3
클래식 사진
속성역할
aspect-ratio요소(박스)의 가로세로 비율을 고정. width만 있어도 height 자동 계산.
object-fit: cover고정된 박스 안에 이미지 내용물을 찌그러짐 없이 채움 (넘치는 부분은 잘림).
기억할 것 — aspect-ratio는 틀을 고정하고, object-fit은 그 틀 안에 내용물을 어떻게 배치할지 정합니다. 카드형 UI에서 이미지 원본 크기가 제각각일 때는 이 두 속성을 항상 함께 씁니다.
