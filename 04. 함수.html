04. 함수

코드 묶음에 이름을 붙여 재사용하기 — 선언 방법 3가지와 호이스팅

🎯 학습 목표

  • ☐ 함수의 구조(이름·매개변수·본문·반환값)를 설명할 수 있다
  • ☐ 선언적 함수 · 익명 함수 · 화살표 함수 · 즉시실행함수를 구분해서 작성할 수 있다
  • ☐ 매개변수 기본값과 return의 동작 규칙을 설명할 수 있다
  • ☐ 호이스팅 때문에 생기는 실행 순서 차이를 예측할 수 있다
  • ☐ 함수를 값처럼 다른 함수에 전달(콜백)할 수 있다
04-1

함수란

함수는 소스코드의 집합으로, 메서드·모듈·기능·프로시저 등으로도 불리며 자바스크립트에서는 function 자료형입니다. 같은 코드를 이름 하나로 몇 번이고 재사용할 수 있게 해줍니다.

function 함수명(매개변수1, 매개변수2) {   // ① 이름, ② 매개변수(입력)
  // ③ 함수 본문 — 실행할 코드
  return 결과값;                        // ④ 반환값(출력, 생략 가능)
}
구성 요소역할
함수명함수를 부를 때 쓰는 이름. 동사로 시작하는 것이 관례 (getTotal, makeCard 등)
매개변수(parameter)함수가 입력으로 받을 값의 이름표. 개수는 0개 이상 자유
함수 본문중괄호 { } 안, 실제로 실행되는 코드
반환값(return value)함수가 결과로 돌려주는 값. 없어도 되고(undefined), 있어도 됨

정리하면 함수는 「입력(매개변수) → 처리(본문) → 출력(반환값)」 구조입니다. 04-4에서 자세히 다룹니다.

04-2

함수 선언 3가지 방법

① 선언적 함수(function declaration) — function 키워드에 함수 이름을 붙여 작성

function greet(name) {
  return `안녕하세요, ${name}님!`;
}
console.log(greet("지민"));
실행 결과
안녕하세요, 지민님!

② 익명 함수(함수 표현식, function expression) — 이름 없는 함수를 변수에 대입. 콜백 형태로 자주 사용

const greet2 = function (name) {
  return `반갑습니다, ${name}님!`;
};
console.log(greet2("수현"));

③ 즉시실행함수(IIFE, Immediately Invoked Function Expression) — 정의하자마자 스스로 실행. 다시 호출할 필요 없는 일회성 초기화 코드에 씁니다.

(function () {
  console.log("페이지 초기화 완료");
})();
🙋 학생들이 자주 틀리는 부분 — IIFE 괄호를 빼먹는 문법 오류
❌ 이렇게 쓰면 안 됩니다 
function () {              // 앞에 감싸는 ( 가 없음!
  console.log("초기화");
}();
// SyntaxError: Function statements require a function name
✅ 이렇게 고칩니다 
(function () {             // 전체를 괄호로 감싸야 "즉시 실행할 식"으로 인식됨
  console.log("초기화");
})();

자바스크립트는 function으로 줄이 시작하면 기본적으로 이름이 있는 선언문으로 해석하려고 합니다. 괄호로 통째로 감싸야 "이건 즉시 실행할 값(식)이다"라고 알려줄 수 있습니다.

04-3

화살표 함수 (Arrow Function)

ES6부터 생긴 더 짧게 쓰는 익명 함수 문법입니다. index.html의 기존 스크립트도, 앞으로 쓸 forEach/map/filter의 콜백도 대부분 이 문법을 씁니다. 매개변수 개수와 본문 줄 수에 따라 생략 규칙이 다릅니다.

상황문법
매개변수 1개name => 코드 (괄호 생략 가능)
매개변수 0개 또는 2개 이상() => 코드 / (a, b) => 코드 (괄호 필수)
본문이 식 하나뿐(a, b) => a + b (중괄호·return 생략, 결과가 자동 반환)
본문이 여러 줄(a, b) => { ...; return 결과; } (중괄호와 return 모두 필요)
const greet3 = (name) => `안녕, ${name}`;      // 매개변수 1개, 식 1개 — 가장 짧은 형태
const add = (a, b) => a + b;                   // 매개변수 2개, 식 1개
const sayHi = () => console.log("hi");          // 매개변수 0개

const makeBadge = (rating) => {                 // 본문이 여러 줄이면 중괄호+return 필수
  const symbol = rating >= 9 ? "🏆" : "⭐";
  return `${symbol} ${rating}`;
};
🙋 학생들이 자주 틀리는 부분 — 여러 줄 화살표 함수에서 return을 빼먹음
❌ 이렇게 쓰면 안 됩니다 
const makeBadge = (rating) => {
  const symbol = rating >= 9 ? "🏆" : "⭐";
  `${symbol} ${rating}`;        // return이 없음!
};

console.log(makeBadge(9.5));    // undefined ❌
✅ 이렇게 고칩니다 
const makeBadge = (rating) => {
  const symbol = rating >= 9 ? "🏆" : "⭐";
  return `${symbol} ${rating}`; // return 붙임
};

console.log(makeBadge(9.5));    // "🏆 9.5" ✅

중괄호 { }를 쓰는 순간 "본문이 여러 줄"로 취급되어, 자동 반환이 사라지고 return을 직접 써야 합니다. 반대로 중괄호 없이 식 하나만 쓰면 그 결과가 자동으로 반환됩니다 — 04-2의 예제들이 그 경우입니다.

⚠️ 화살표 함수로 객체를 바로 반환하려면 괄호로 한 번 더 감싸야 함

() => { title: "영화" }라고 쓰면 자바스크립트가 { }를 함수 본문으로 오해합니다. 객체를 즉시 반환하려면 () => ({ title: "영화" })처럼 객체 전체를 소괄호로 한 번 더 감싸야 합니다.

04-4

함수 호출 / 매개변수와 return

함수는 반드시 선언(정의)되어야 실행할 수 있고, 호출하는 것을 함수 실행이라 합니다. return 값이 있다면 받을 변수를 준비합니다.

함수명();                    // return 없는 경우
const result = 함수명(인자1, 인자2); // return 있는 경우
규칙동작
인자를 더 많이 넘기면초과분은 무시됨 (오류 아님)
인자를 더 적게 넘기면빠진 매개변수는 undefined가 됨
return 값을 지정하지 않으면undefined가 반환됨
return을 만나면즉시 함수를 종료하고 호출한 곳으로 돌아감
function add(a, b) { return a + b; }
console.log(add(1, 2, 99)); // 3        (99는 무시)
console.log(add(1));        // NaN      (b가 undefined → 1 + undefined)

return이 아예 없거나, return만 쓰고 값을 안 적으면 함수는 undefined를 반환합니다. "함수를 실행은 했지만 돌려줄 값을 안 정했다"는 뜻입니다.

function logMovie(title) {
  console.log("영화:", title);   // 화면에 출력만 하고
  // return이 없음!
}

const result = logMovie("도시의 밤");
console.log(result);   // undefined — 함수는 실행됐지만 돌려준 값이 없음
실행 결과
영화: 도시의 밤
undefined
🙋 학생들이 자주 틀리는 부분 — console.log만 하는 함수의 결과를 변수에 담으려는 실수
❌ 이렇게 쓰면 안 됩니다 
function getDiscountPrice(price) {
  const discounted = price * 0.9;
  console.log(discounted);   // 화면에 찍기만 하고 return을 안 씀!
}

const finalPrice = getDiscountPrice(10000);
console.log(`최종 가격: ${finalPrice}원`);
// "최종 가격: undefined원"  ← 화면에는 9000이 찍혔지만, 함수 밖에서는 못 씀!
✅ 이렇게 고칩니다 
function getDiscountPrice(price) {
  const discounted = price * 0.9;
  return discounted;         // 값을 밖으로 돌려줌
}

const finalPrice = getDiscountPrice(10000);
console.log(`최종 가격: ${finalPrice}원`);
// "최종 가격: 9000원"  ✅

console.log()는 화면에 보여주기만 할 뿐, 그 값을 함수 밖으로 전달하지는 않습니다. 함수의 결과값을 다른 곳에서 계속 쓰고 싶다면 반드시 return으로 돌려줘야 합니다 — "찍었으니까 됐다"는 이 과정에서 가장 흔한 초보 실수 중 하나입니다.

04-5

매개변수 기본값 (Default Parameters)

ES6부터는 매개변수 자리에 = 값을 붙여, 인자를 안 넘겼을 때 자동으로 쓰일 기본값을 정할 수 있습니다.

function getAgeRating(title, rating = "전체 관람가") {
  return `${title} - ${rating}`;
}

console.log(getAgeRating("스텔라 오디세이"));
// "스텔라 오디세이 - 전체 관람가"  (rating을 안 넘겨서 기본값 사용)

console.log(getAgeRating("어둠의 미로", "15세 이상"));
// "어둠의 미로 - 15세 이상"  (넘긴 값으로 덮어씀)

04-4에서 배웠듯, 인자를 안 넘기면 원래는 undefined가 됩니다. 기본값은 정확히는 "매개변수가 undefined일 때만" 적용됩니다.

function greet(name = "손님") {
  return `안녕하세요, ${name}님!`;
}

console.log(greet());          // "안녕하세요, 손님님!"  (undefined → 기본값)
console.log(greet(undefined)); // "안녕하세요, 손님님!"  (명시적으로 undefined를 넘겨도 기본값!)
console.log(greet(null));      // "안녕하세요, null님!"  (null은 기본값 적용 안 됨 — 02-3 참고)
⚠️ 기본값은 undefined에만 반응, null에는 반응 안 함

02-3에서 배운 null(의도적 없음)과 undefined(자동 없음)의 차이가 여기서도 그대로 적용됩니다. null을 넘기면 "진짜 null값을 쓰겠다"는 의도로 보고 기본값을 적용하지 않습니다.

04-6

가변인자 함수 — arguments와 나머지 매개변수

매개변수 개수가 정해지지 않은 함수를 가변인자 함수라고 합니다. function 키워드로 만든 함수 내부에는 arguments라는 유사 배열 객체가 자동 생성되어 전달된 인자가 순서대로 저장됩니다.

function sum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}
console.log(sum(1, 2, 3));       // 6
console.log(sum(10, 20, 30, 40)); // 100
⚠️ arguments는 '유사' 배열

length와 인덱스는 있지만 진짜 배열이 아니라서 push, sort 같은 배열 메서드는 쓸 수 없습니다. 또 화살표 함수에는 arguments가 없습니다.

최신 코드에서는 나머지 매개변수(rest parameters) ...이름을 씁니다 — 화살표 함수에서도 쓸 수 있고, 진짜 배열이라 배열 메서드를 바로 쓸 수 있습니다.

const sum2 = (...nums) => {         // 화살표 함수에서도 사용 가능!
  return nums.reduce((acc, n) => acc + n, 0);
};
console.log(sum2(1, 2, 3));         // 6
console.log(Array.isArray(sum2));   // nums는 진짜 배열이라 sort, map 등 다 됨
04-7

함수를 매개변수로 전달 — 콜백

함수도 하나의 자료형(값)이므로 다른 함수의 매개변수로 전달할 수 있습니다. 전달받은 쪽에서 나중에 호출한다고 해서 콜백(callback) 함수라고 부릅니다.

function run(callback) {
  console.log("작업 시작");
  callback();              // 전달받은 함수를 호출
}

run(function () {
  console.log("전달된 함수 실행!");
});
실행 결과
작업 시작
전달된 함수 실행!
🙋 학생들이 자주 틀리는 부분 — 함수를 '전달'하는 대신 실수로 '실행'해버림
❌ 이렇게 쓰면 안 됩니다 
function run(callback) {
  console.log("작업 시작");
  callback();
}

function sayDone() {
  console.log("완료!");
}

run(sayDone());   // 괄호를 붙임! — sayDone을 그 자리에서 바로 실행해버림
// "완료!"가 run() 호출도 하기 전에 먼저 출력되고,
// run에는 sayDone의 리턴값(undefined)이 전달되어 callback()에서 오류 발생
✅ 이렇게 고칩니다 
run(sayDone);      // 괄호 없이 이름만! — "이 함수 자체"를 전달
// "작업 시작" → "완료!" 순서로 정상 출력

sayDone(괄호 없음)은 함수 자체를 가리키는 참조이고, sayDone()(괄호 있음)은 그 자리에서 즉시 실행하고 결과값만 남기는 것입니다. 콜백으로 전달할 때는 괄호를 붙이지 않는 것이 원칙입니다 — 실행은 전달받은 쪽(run 내부)이 원하는 시점에 하도록 맡기는 것이니까요.

⭐ 이미 콜백을 쓰고 있습니다

3장의 sort(function(a,b){...}), forEach(function(item){...}), 그리고 9장의 addEventListener('click', function(){...}) — 전부 콜백입니다. 자바스크립트 전체를 관통하는 개념입니다.

04-8

내장 함수

isNaN()은 값이 NaN인지 확인합니다. NaN은 NaN === NaN조차 false가 나오는 특이한 값이라(02-9 참고), 반드시 이 함수로 판별해야 합니다.

console.log(isNaN(10));        // false (숫자 맞음)
console.log(isNaN("가나다"));   // true  (숫자로 변환 불가)
console.log(NaN === NaN);      // false (비교연산자로는 판별 불가!)
console.log(isNaN(NaN));       // true  (isNaN으로는 정상 판별)

isFinite()는 값이 유한한 숫자인지 확인합니다 (무한대·NaN이 아니면 true).

console.log(isFinite(100));      // true
console.log(isFinite(Infinity)); // false
console.log(isFinite("가나다"));  // false (NaN으로 변환됨)
⚠️ eval()은 사용 금지 수준의 레거시

eval(문자열)은 문자열을 코드로 실행하는 함수지만, 보안 위험(악성 코드 실행 가능)과 성능 문제로 실무에서 쓰지 않습니다. "이런 게 있었다" 정도만 알아두세요.

04-9

호이스팅(Hoisting) — 선언이 끌어올려진다

자바스크립트는 코드를 실행하기 전에 선언부를 먼저 훑어 메모리에 등록합니다. 그래서 마치 선언이 파일 맨 위로 끌어올려진 것처럼 동작하는데, 이를 호이스팅이라고 합니다. 중요한 것은 선언 방식에 따라 결과가 다르다는 점입니다.

hello();          // 동작! — 선언적 함수는 "함수 전체"가 호이스팅
function hello() {
  console.log("안녕하세요");
}

bye();            // TypeError: bye is not a function
var bye = function () {
  console.log("안녕히가세요");
};
// var bye "선언"만 끌어올려져 undefined 상태 → undefined() 호출이라 오류
선언 방식호이스팅 결과선언 전 사용하면
function 선언적 함수함수 전체가 올라감정상 동작
var 변수선언만 올라감 (값은 undefined)undefined
let / const올라가지만 초기화 전 접근 금지(TDZ)ReferenceError
화살표 함수 (const 대입)let/const와 동일하게 TDZ 적용ReferenceError
⭐ 호이스팅에 기대지 말 것

선언적 함수가 '아래에 있어도 동작'하는 건 알아두되, 코드는 항상 「선언 먼저, 사용 나중」 순서로 작성하는 습관을 들이세요. let/const가 선언 전 사용을 오류로 막아주는 이유가 바로 이것입니다.
