03. 배열

여러 값을 순서대로 담는 그릇 — 영화 6편을 데이터로 다루는 첫 걸음

🎯 학습 목표

  • ☐ 배열이 왜 필요한지 설명하고, 선언·초기화할 수 있다
  • ☐ 인덱스로 값을 읽고 쓰며, 배열이 참조 자료형임을 이해한다
  • ☐ 추가/제거/검색/자르기 메서드를 상황에 맞게 골라 쓸 수 있다
  • ☐ sort()에 콜백 함수를 넣어 원하는 기준으로 정렬할 수 있다
  • ☐ for / forEach로 배열을 순회할 수 있다
03-1

배열이란

02장에서 영화 정보를 변수 하나에 담았습니다. 그런데 영화가 6편이면 title1, title2, title3...처럼 변수를 6개 만들어야 할까요? 배열(Array)은 이런 상황을 위해 여러 값을 순서대로 하나의 묶음에 담는 자료형입니다.

// 배열 없이 — 영화가 늘어날 때마다 변수가 계속 늘어남 (비효율적)
const title1 = "도시의 밤";
const title2 = "파랑의 계절";
const title3 = "스텔라 오디세이";

// 배열로 — 몇 편이든 변수 하나로 관리
const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];

배열은 02-3에서 배운 참조 자료형입니다 — 자료형 제한이 없어 숫자·문자열·불리언·객체·함수까지 무엇이든 담을 수 있습니다.

03-2

배열 선언과 초기화

// [ ] 리터럴 방식 — 생성과 초기화를 동시에 (권장)
const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];

// new Array() 방식
const arr1 = new Array();        // 빈 배열
const arr2 = new Array(5);       // 길이 5짜리 빈 배열
const arr3 = new Array(1, 2, 3); // [1, 2, 3]
⚠️ new Array(숫자 하나)의 함정

new Array(5)는 [5]가 아니라 길이가 5인 빈 배열을 만듭니다. 헷갈림을 피하려면 항상 [ ] 리터럴을 쓰세요.

03-3

인덱스로 값 읽고 쓰기

배열의 각 칸에는 0부터 시작하는 번호(인덱스)가 붙습니다. 번호 범위는 0 ~ (길이-1)입니다.

const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];

console.log(titles[0]);        // 첫 번째 값 읽기
console.log(titles.length);    // 배열 길이
console.log(titles[titles.length - 1]); // 마지막 값

titles[1] = "여름의 문";       // 값 바꾸기
실행 결과
도시의 밤
3
스텔라 오디세이
🙋 학생들이 자주 틀리는 부분 — 존재하지 않는 인덱스 접근
❌ 이렇게 쓰면 안 됩니다 
const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"]; // length = 3

console.log(titles[3]);           // undefined (인덱스 0,1,2까지만 있음)
console.log(titles[3].length);    // TypeError! undefined는 length가 없음
✅ 이렇게 고칩니다 
const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];

const target = titles[3];
if (target !== undefined) {       // 사용 전에 존재 여부부터 확인
  console.log(target.length);
} else {
  console.log("해당 인덱스에 값이 없습니다");
}

범위를 벗어난 인덱스는 오류 없이 조용히 undefined를 돌려줍니다. 문제는 그 undefined에 또 무언가(.length, .title 등)를 시도할 때 터진다는 것 — 배열 인덱스 관련 오류는 대부분 실제 원인이 몇 줄 위에 있습니다.

03-4

배열은 참조 자료형 — 복사가 아니라 같은 걸 가리킴

02-3에서 배운 원시형/참조형 구분을 배열에 적용해봅니다. 배열을 다른 변수에 대입하면 복사되는 게 아니라, 같은 배열을 함께 가리키게 됩니다.

const original = ["도시의 밤", "파랑의 계절"];
const copy = original;         // 복사가 아니라 "같은 배열을 가리키는 이름표"

copy.push("스텔라 오디세이");
console.log(original);   // ["도시의 밤", "파랑의 계절", "스텔라 오디세이"]
// ↑ copy만 바꿨는데 original도 함께 바뀜! (원시형이었다면 이런 일 없음)

진짜 복사본이 필요하면 스프레드 연산자(...)를 씁니다 (05장에서 자세히 배웁니다).

const original = ["도시의 밤", "파랑의 계절"];
const realCopy = [...original];   // 새로운 배열로 복사

realCopy.push("스텔라 오디세이");
console.log(original); // ["도시의 밤", "파랑의 계절"]  ← 안 바뀜! ✅
03-5

요소 추가·제거 — push / pop / shift / unshift

배열의 양 끝에 요소를 추가하거나 제거하는 4개 메서드입니다. 전부 원본 배열을 직접 바꿉니다.

메서드위치동작리턴값
push(값)맨 뒤추가바뀐 후의 새 길이
pop()맨 뒤제거제거된 값
unshift(값)맨 앞추가바뀐 후의 새 길이
shift()맨 앞제거제거된 값
const queue = ["도시의 밤", "파랑의 계절"];

const newLength = queue.push("스텔라 오디세이");   // 맨 뒤에 추가
console.log(queue);        // ["도시의 밤", "파랑의 계절", "스텔라 오디세이"]
console.log(newLength);    // 3  (push는 추가한 값이 아니라 "새 길이"를 리턴!)

const removed = queue.shift();   // 맨 앞 제거
console.log(removed);      // "도시의 밤"  (제거된 값을 리턴)
console.log(queue);        // ["파랑의 계절", "스텔라 오디세이"]
🙋 학생들이 자주 틀리는 부분 — push의 리턴값을 추가된 배열이라고 착각
❌ 이렇게 쓰면 안 됩니다 
const movies = ["도시의 밤"];
const updated = movies.push("파랑의 계절");

console.log(updated);   // 2  ← 배열이 아니라 숫자! (당황스러운 결과)
updated.forEach(...)     // TypeError: updated.forEach is not a function
✅ 이렇게 고칩니다 
const movies = ["도시의 밤"];
movies.push("파랑의 계절");   // 리턴값을 안 쓰고, movies 자체를 사용

console.log(movies);    // ["도시의 밤", "파랑의 계절"] ✅
movies.forEach(...)      // 정상 동작

push/unshift는 추가된 새 배열이 아니라 '새 길이(숫자)'를 리턴합니다. 배열 자체가 필요하면 리턴값을 쓰지 말고, push를 실행한 원본 변수(movies)를 그대로 사용하세요.

03-6

배열에서 값 찾기 — indexOf / includes

메서드설명값이 없을 때
indexOf(값)값이 위치한 인덱스를 리턴-1
includes(값)값이 있는지 없는지 true/false로 리턴false
const genres = ["SF", "액션", "드라마"];

console.log(genres.indexOf("액션"));   // 1
console.log(genres.indexOf("로맨스")); // -1 (없음)

console.log(genres.includes("SF"));     // true
console.log(genres.includes("로맨스")); // false
⚠️ indexOf 결과를 if 조건에 그냥 쓰면 안 됩니다

if (genres.indexOf("SF"))처럼 쓰면, indexOf가 0(첫 번째 인덱스)을 리턴했을 때도 falsy라서 "없다"로 잘못 판단합니다(02-13 falsy 참고). 존재 여부만 확인할 땐 indexOf(...) !== -1 또는 더 명확한 includes()를 쓰세요.

03-7

배열 합치기·자르기 — concat / slice / splice

concat(배열) — 두 배열을 합친 새 배열을 리턴합니다 (원본 유지).

const part1 = ["도시의 밤", "파랑의 계절"];
const part2 = ["스텔라 오디세이"];
const merged = part1.concat(part2);
console.log(merged);   // ["도시의 밤", "파랑의 계절", "스텔라 오디세이"]
console.log(part1);    // ["도시의 밤", "파랑의 계절"]  ← 원본 그대로

slice(시작, 끝) — 일부를 잘라 새 배열로 리턴합니다 (끝 인덱스는 포함 안 됨, 원본 유지).

splice(위치, 개수, 추가값...) — 원본 배열을 직접 잘라내거나 끼워 넣습니다.

const list = ["a", "b", "c", "d"];
console.log(list.slice(1, 3));  // ["b", "c"]  (원본 유지)
console.log(list);              // ["a", "b", "c", "d"]

list.splice(1, 2);              // 인덱스 1부터 2개 제거 (원본 변경!)
console.log(list);              // ["a", "d"]

list.splice(1, 0, "X", "Y");    // 인덱스 1에 0개 제거하고 "X","Y" 끼워넣기
console.log(list);              // ["a", "X", "Y", "d"]
⭐ slice vs splice — 이름은 비슷, 동작은 정반대

slice는 원본을 그대로 두고 복사본을 리턴하고, splice는 원본을 직접 수술합니다. 's'가 하나 더 있는 쪽(splice)이 더 침습적이라고 기억하면 헷갈리지 않습니다.

03-8

정렬과 뒤집기 — reverse / sort

reverse()는 배열 순서를 그대로 뒤집습니다 (원본 변경). sort()는 기본적으로 문자열 기준 오름차순으로 정렬해서, 숫자를 그냥 정렬하면 이상한 결과가 나옵니다.

const nums = [30, 4, 100];
console.log(nums.sort());           // [100, 30, 4] ?! (문자열 "1","3","4" 순)

정렬 방법을 바꾸려면 sort()의 인자에 비교 함수를 넣습니다. 두 매개변수(left, right)를 받아, 음수를 리턴하면 left가 앞으로 갑니다.

const ratings = [9.2, 8.7, 9.8, 9.0];

ratings.sort(function (left, right) {
  return left - right;   // 오름차순
});

ratings.sort(function (left, right) {
  return right - left;   // 내림차순
});
console.log(ratings);
실행 결과
[9.8, 9.2, 9, 8.7]
🙋 학생들이 자주 틀리는 부분 — sort로 원본이 바뀌어서 다른 곳이 망가짐
❌ 이렇게 쓰면 안 됩니다 
const movies = [
  { title: "은하수 특급", rating: 9.6 },
  { title: "도시의 밤", rating: 9.8 },
];

function showRanking() {
  movies.sort((a, b) => b.rating - a.rating);  // 원본을 직접 정렬!
  console.log("랭킹:", movies.map(m => m.title));
}

function showOriginalOrder() {
  console.log("원래 순서:", movies.map(m => m.title));
}

showRanking();
showOriginalOrder();  // "원래 순서"가 이미 랭킹 순서로 뒤바뀌어버림!
✅ 이렇게 고칩니다 
function showRanking() {
  const sorted = [...movies].sort((a, b) => b.rating - a.rating); // 복사본만 정렬
  console.log("랭킹:", sorted.map(m => m.title));
}

function showOriginalOrder() {
  console.log("원래 순서:", movies.map(m => m.title));  // movies는 그대로 ✅
}

sort는 원본을 직접 바꾸기 때문에, 같은 배열을 여러 화면(랭킹 테이블, 카드 목록 등)에서 서로 다른 순서로 써야 할 때 문제가 됩니다. 정렬이 필요할 때마다 [...배열]로 복사본을 만들고 그 복사본만 정렬하는 것을 습관으로 삼으세요.

03-9

배열을 문자열로 — join / toString

join(구분자)는 배열 요소를 원하는 구분자로 이어붙인 문자열을 리턴합니다. toString()은 항상 쉼표(,)로만 이어붙입니다.

const genres = ["SF", "액션", "모험"];

console.log(genres.join(" · "));   // "SF · 액션 · 모험"
console.log(genres.join(""));      // "SF액션모험"  (구분자 없이 붙임)
console.log(genres.toString());    // "SF,액션,모험" (항상 쉼표)
03-10

배열 순회 — for · for...in · forEach

const titles = ["도시의 밤", "파랑의 계절", "스텔라 오디세이"];

// ① 기본 for문 — 인덱스가 필요할 때
for (let i = 0; i < titles.length; i++) {
  console.log(i + "위 : " + titles[i]);
}

// ② for...in — 인덱스(키)를 순회
for (const i in titles) {
  console.log(titles[i]);
}

// ③ forEach — 요소를 하나씩 콜백으로 전달 (실무에서 가장 많이)
titles.forEach(function (title, i) {
  console.log(i, title);
});
⭐ 배열 순회는 forEach(또는 기본 for)가 기본

for...in은 원래 객체용이라(5장) 배열에서는 예상 밖 동작이 생길 수 있습니다. 배열에는 forEach / for / for...of를 쓰는 게 안전합니다.

⚠️ forEach는 새 배열을 만들지 않습니다

const result = titles.forEach(...)처럼 결과를 담으려 하면 result는 undefined입니다. forEach는 '순회하며 무언가 실행'만 할 뿐 값을 돌려주지 않습니다 — 가공된 새 배열이 필요하면 04장 이후에 배울 map을 씁니다.
